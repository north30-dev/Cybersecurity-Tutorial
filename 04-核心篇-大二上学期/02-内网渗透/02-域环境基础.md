# 域环境基础

## 1. Active Directory架构概述

### 1.1 什么是Active Directory

Active Directory（活动目录，简称AD）是Microsoft开发的目录服务，用于Windows域网络中的集中身份认证和资源管理。它是内网环境中最重要的基础设施之一，也是内网渗透的核心目标。

**AD的核心功能：**
- 集中存储和管理网络资源信息
- 提供统一的身份认证和授权服务
- 实现组策略的集中管理
- 支持域间信任关系
- 提供轻量级目录访问协议（LDAP）服务

### 1.2 AD逻辑结构

Active Directory采用层次化的逻辑结构：

```
林（Forest）
└── 域树（Domain Tree）
    └── 域（Domain）
        ├── 组织单位（OU）
        │   ├── 子OU
        │   └── 对象（用户、计算机、组等）
        └── 容器（Container）
```

**各层级说明：**

| 层级 | 说明 | 示例 |
|------|------|------|
| 林（Forest） | AD的最高层级，一个或多个域树的集合 | corp.local |
| 域树（Domain Tree） | 共享连续命名空间的域层次结构 | corp.local → sub.corp.local |
| 域（Domain） | 安全边界和管理单元 | corp.local |
| 组织单位（OU） | 组织和管理对象的容器 | OU=IT, DC=corp, DC=local |

### 1.3 AD物理结构

AD的物理结构主要包括：

**域控制器（Domain Controller，DC）：**
- 存储AD数据库的副本
- 提供认证和授权服务
- 多个DC之间通过复制保持数据同步

**站点（Site）：**
- 定义网络拓扑结构
- 控制复制流量和客户端登录行为
- 通常对应物理网络位置

**AD数据库文件：**
```
C:\Windows\NTDS\
├── ntds.dit          # AD数据库主文件
├── edb.log           # 事务日志
├── edb.chk           # 检查点文件
└── temp.edb          # 临时工作文件
```

### 1.4 命名规范

AD使用LDAP可分辨名称（DN）来唯一标识对象：

```
CN=张三,OU=IT部门,OU=员工,DC=corp,DC=local
```

**命名属性说明：**
- **DC** - Domain Component（域组件）
- **OU** - Organizational Unit（组织单位）
- **CN** - Common Name（通用名称）
- **DN** - Distinguished Name（可分辨名称）
- **RDN** - Relative Distinguished Name（相对可分辨名称）

## 2. 域控制器与域成员

### 2.1 域控制器（Domain Controller）

域控制器是域环境中的核心服务器，负责：

**主要职责：**
- 存储和维护AD数据库
- 处理用户认证请求
- 执行组策略
- 提供目录服务查询
- 管理域间信任关系

**域控制器角色：**

**PDC Emulator（主域控制器模拟器）：**
- 域内唯一的PDC模拟器
- 处理密码更改和锁定
- 时间同步权威
- 兼容旧版客户端

**RID Master（RID主机）：**
- 分配RID池给域控制器
- 确保安全主体SID唯一性

**Infrastructure Master（基础结构主机）：**
- 维护组中跨域成员的引用
- 处理对象移动和重命名

**Schema Master（架构主机）：**
- 林级别唯一
- 管理AD架构定义

**Domain Naming Master（域命名主机）：**
- 林级别唯一
- 控制域的添加和删除

### 2.2 域成员类型

**域内计算机分类：**

| 类型 | 说明 | 特点 |
|------|------|------|
| 域控制器 | 运行AD DS服务 | 存储AD数据库副本 |
| 成员服务器 | 加入域的Windows服务器 | 使用域账户认证 |
| 工作站 | 加入域的客户端计算机 | 应用域组策略 |
| 独立服务器 | 未加入域的服务器 | 使用本地账户 |

### 2.3 域加入过程

计算机加入域的流程：

```
1. 客户端定位域控制器（DNS查询SRV记录）
2. 与域控制器建立安全通道
3. 验证加入权限（需要域管理员或委派权限）
4. 创建计算机账户对象
5. 设置安全通道密码
6. 配置组策略应用
7. 重启完成加入
```

**查看域加入状态：**
```powershell
# PowerShell查看域信息
Get-ADDomain

# 查看域控制器
Get-ADDomainController

# 查看计算机账户
Get-ADComputer -Filter * -Properties *
```

## 3. 认证机制

### 3.1 Kerberos认证协议

Kerberos是Windows域环境的默认认证协议，基于票据（Ticket）的信任机制。

**Kerberos核心组件：**
- **KDC（Key Distribution Center）**：密钥分发中心，运行在域控制器上
- **AS（Authentication Service）**：认证服务，发放TGT
- **TGS（Ticket Granting Service）**：票据授予服务，发放服务票据
- **TGT（Ticket Granting Ticket）**：票据授予票据
- **ST（Service Ticket）**：服务票据

**Kerberos认证流程：**

```
┌─────────┐                    ┌─────────┐                    ┌─────────┐
│  Client │                    │   KDC   │                    │  Server │
└────┬────┘                    └────┬────┘                    └────┬────┘
     │                              │                              │
     │  1. AS-REQ (用户名,时间戳)   │                              │
     │ ─────────────────────────────>│                              │
     │                              │                              │
     │  2. AS-REP (TGT,Session Key) │                              │
     │ <─────────────────────────────│                              │
     │                              │                              │
     │  3. TGS-REQ (TGT,SPN)        │                              │
     │ ─────────────────────────────>│                              │
     │                              │                              │
     │  4. TGS-REP (Service Ticket) │                              │
     │ <─────────────────────────────│                              │
     │                              │                              │
     │  5. AP-REQ (Service Ticket)  │                              │
     │ ─────────────────────────────────────────────────────────────>│
     │                              │                              │
     │  6. AP-REP (认证确认)        │                              │
     │ <─────────────────────────────────────────────────────────────│
     │                              │                              │
```

**详细步骤说明：**

**步骤1-2：获取TGT**
```
客户端发送: AS-REQ
- 用户Principal（如：user@CORP.LOCAL）
- 时间戳（用用户密码哈希加密）

KDC响应: AS-REP
- TGT（用krbtgt账户密码哈希加密）
- Session Key（用用户密码哈希加密）
```

**步骤3-4：获取服务票据**
```
客户端发送: TGS-REQ
- TGT
- 目标服务SPN
- 认证器（用Session Key加密）

KDC响应: TGS-REP
- Service Ticket（用服务账户密码哈希加密）
- 新的Session Key
```

**步骤5-6：访问服务**
```
客户端发送: AP-REQ
- Service Ticket
- 认证器（用服务Session Key加密）

服务器验证后建立会话
```

**Kerberos配置查看：**
```powershell
# 查看Kerberos策略
net accounts /domain

# 查看当前Kerberos票据
klist

# 清除Kerberos票据
klist purge

# 请求新的票据
klist request
```

### 3.2 NTLM认证协议

NTLM（NT LAN Manager）是Windows的旧版认证协议，在某些场景下仍被使用。

**NTLM认证流程：**

```
┌─────────┐                    ┌─────────┐
│  Client │                    │  Server │
└────┬────┘                    └────┬────┘
     │                              │
     │  1. Type 1 (Negotiate)       │
     │ ─────────────────────────────>│
     │                              │
     │  2. Type 2 (Challenge)       │
     │ <─────────────────────────────│
     │    随机Challenge值           │
     │                              │
     │  3. Type 3 (Authenticate)    │
     │ ─────────────────────────────>│
     │    NTLM Response             │
     │                              │
```

**NTLM响应类型：**

| 类型 | 说明 | 安全性 |
|------|------|--------|
| LM Response | LAN Manager响应 | 极低，已弃用 |
| NTLM v1 Response | NTLM版本1响应 | 低，易被破解 |
| NTLM v2 Response | NTLM版本2响应 | 中等，推荐使用 |

**NTLM使用场景：**
- 域控制器不可用时
- 跨域森林认证
- 某些旧版应用
- 工作组环境

### 3.3 认证协议对比

| 特性 | Kerberos | NTLM |
|------|----------|------|
| 安全性 | 高 | 中 |
| 性能 | 高（票据缓存） | 低（每次认证） |
| 双向认证 | 支持 | 不支持 |
| 委派 | 支持 | 不支持 |
| 单点登录 | 支持 | 不支持 |
| 网络环境 | 域环境 | 域/工作组 |

## 4. 域信任关系

### 4.1 信任关系类型

**信任方向：**
- **单向信任**：域A信任域B，域B不信任域A
- **双向信任**：域A和域B相互信任

**信任传递性：**
- **可传递信任**：A信任B，B信任C，则A信任C
- **不可传递信任**：信任关系不会延伸

**常见信任类型：**

| 信任类型 | 方向 | 传递性 | 用途 |
|----------|------|--------|------|
| 父子信任 | 双向 | 可传递 | 同一域树内的父子域 |
| 树根信任 | 双向 | 可传递 | 同一林内的域树之间 |
| 快捷信任 | 双向 | 可传递 | 优化跨域认证路径 |
| 外部信任 | 单向/双向 | 不可传递 | 不同林之间 |
| 林信任 | 单向/双向 | 可传递 | 不同林之间 |

### 4.2 信任关系查看

```powershell
# 查看域信任关系
Get-ADTrust -Filter *

# 查看特定信任关系
Get-ADTrust -Identity "trusted.domain.local"

# 使用nltest查看信任
nltest /domain_trusts

# 验证信任关系
nltest /verify:trusted.domain.local
```

### 4.3 SID History与跨域访问

SID History允许用户在跨域迁移后保留原有权限：

```powershell
# 查看用户SID History
Get-ADUser user1 -Properties sIDHistory

# SID History安全风险
# 攻击者可能利用SID History获取额外权限
```

## 5. 组策略基础

### 5.1 组策略概述

组策略（Group Policy）是AD中实现集中配置管理的重要机制。

**组策略对象（GPO）结构：**
```
GPO
├── GPC（Group Policy Container）- AD中的配置信息
└── GPT（Group Policy Template）- SYSVOL中的策略文件
```

**GPO存储位置：**
```
# AD中的GPC
CN=Policies,CN=System,DC=corp,DC=local

# SYSVOL中的GPT
\\corp.local\SYSVOL\corp.local\Policies\{GPO-GUID}
```

### 5.2 组策略应用机制

**GPO链接优先级（从高到低）：**
1. 强制（Enforced）
2. OU级别
3. 域级别
4. 站点级别

**继承规则：**
- 子容器继承父容器GPO
- 可阻止继承（Block Inheritance）
- 可强制应用（Enforce）

**GPO处理顺序：**
```
LSDOU原则：
1. Local（本地策略）
2. Site（站点策略）
3. Domain（域策略）
4. OU（组织单位策略，从父到子）
```

### 5.3 常用组策略设置

**安全设置：**
```
计算机配置 → Windows设置 → 安全设置
├── 账户策略
│   ├── 密码策略
│   └── 账户锁定策略
├── 本地策略
│   ├── 审核策略
│   ├── 用户权限分配
│   └── 安全选项
└── 受限组
```

**密码策略示例：**
```powershell
# 查看域密码策略
Get-ADFineGrainedPasswordPolicy -Filter *

# 查看默认密码策略
net accounts /domain
```

### 5.4 组策略管理命令

```powershell
# 查看所有GPO
Get-GPO -All

# 查看特定GPO
Get-GPO -Name "Default Domain Policy"

# 查看GPO链接
Get-GPOReport -Name "Default Domain Policy" -ReportType HTML -Path "C:\gpo.html"

# 创建新GPO
New-GPO -Name "New Policy" -Comment "新策略"

# 链接GPO到OU
New-GPLink -Name "New Policy" -Target "OU=IT,DC=corp,DC=local"

# 备份GPO
Backup-GPO -Name "Default Domain Policy" -Path "C:\GPOBackup"
```

### 5.5 组策略安全风险

**常见攻击面：**
- SYSVOL中的敏感信息（如密码）
- GPO中的计划任务配置
- 启动脚本中的凭证
- 受限组配置

**查看SYSVOL：**
```powershell
# 访问SYSVOL
dir \\corp.local\SYSVOL\corp.local\

# 搜索可能的敏感文件
findstr /S /I "password" \\corp.local\SYSVOL\corp.local\Policies\*.xml
```

## 6. 实战环境搭建建议

### 6.1 实验环境拓扑

建议搭建以下实验环境：

```
                    Internet
                        │
                    防火墙

    DC01 (Win2019)  WEB01 (Win2016)  CLIENT01 (Win10)
    192.168.1.10     192.168.1.20      192.168.1.100
        │
    域控制器
    AD DS + DNS
```

### 6.2 域环境搭建步骤

**安装域控制器：**
```powershell
# 安装AD DS角色
Install-WindowsFeature -Name AD-Domain-Services -IncludeManagementTools

# 提升为域控制器
Install-ADDSForest `
    -DomainName "corp.local" `
    -DomainNetbiosName "CORP" `
    -ForestMode "WinThreshold" `
    -DomainMode "WinThreshold" `
    -InstallDns:$true
```

**创建组织结构：**
```powershell
# 创建OU
New-ADOrganizationalUnit -Name "IT部门" -Path "DC=corp,DC=local"
New-ADOrganizationalUnit -Name "HR部门" -Path "DC=corp,DC=local"

# 创建用户
New-ADUser -Name "admin" -SamAccountName "admin" `
    -UserPrincipalName "admin@corp.local" `
    -Enabled $true -AccountPassword (ConvertTo-SecureString "P@ssw0rd!" -AsPlainText -Force)

# 创建组
New-ADGroup -Name "ITAdmins" -SamAccountName "ITAdmins" `
    -GroupCategory Security -GroupScope Global `
    -Path "OU=IT部门,DC=corp,DC=local"
```
