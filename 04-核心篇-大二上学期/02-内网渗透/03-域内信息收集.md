# 域内信息收集

## 1. 域信息收集概述

### 1.1 信息收集的重要性

域内信息收集是内网渗透的关键环节，直接决定了后续攻击路径的选择和成功率。通过全面的信息收集，攻击者可以：

- 了解域环境架构和规模
- 发现高价值目标（域控、管理员账户）
- 识别攻击路径和权限提升机会
- 为横向移动提供情报支持

### 1.2 信息收集原则

**隐蔽性原则：**
- 优先使用系统原生命令
- 避免触发安全设备告警
- 合理控制查询频率
- 使用合法账户权限

**全面性原则：**
- 收集域架构信息
- 收集用户和组信息
- 收集计算机信息
- 收集权限和ACL信息
- 收集凭证信息

### 1.3 信息收集分类

| 类别 | 内容 | 方法 |
|------|------|------|
| 域基础信息 | 域名、域控、功能级别 | net命令、PowerShell |
| 用户信息 | 用户列表、属性、组成员 | LDAP查询、AD模块 |
| 计算机信息 | 计算机列表、操作系统、服务 | DNS、AD查询 |
| 权限信息 | ACL、委派、GPO | PowerView、BloodHound |
| 凭证信息 | 缓存票据、内存凭证 | Mimikatz、Rubeus |

## 2. 原生命令枚举

### 2.1 net命令系列

net命令是Windows系统自带的网络管理命令，无需额外工具即可收集大量域信息。

**域基础信息收集：**

```cmd
# 查看当前域名
net config workstation

# 查看域信息
net view /domain

# 查看域控制器
net time /domain

# 查看域用户列表
net user /domain

# 查看特定用户详细信息
net user administrator /domain

# 查看域组列表
net group /domain

# 查看特定组成员
net group "Domain Admins" /domain
net group "Enterprise Admins" /domain
net group "Domain Controllers" /domain

# 查看本地管理员组
net localgroup administrators
```

**实战技巧：**
```cmd
# 批量获取所有用户信息（保存到文件）
for /f "skip=6 tokens=1" %i in ('net user /domain') do @net user %i /domain >> users.txt

# 查看域计算机
net view

# 查看特定域的计算机
net view /domain:CORP
```

### 2.2 dsquery命令

dsquery是Windows Server自带的目录服务查询工具，功能更强大。

**基本查询：**
```cmd
# 查询所有用户
dsquery user

# 查询所有计算机
dsquery computer

# 查询所有组
dsquery group

# 查询域控制器
dsquery server

# 查询组织单位
dsquery ou

# 查询联系人
dsquery contact
```

**高级查询：**
```cmd
# 查询特定OU下的用户
dsquery user "OU=IT,DC=corp,DC=local"

# 查询禁用的账户
dsquery user -disabled

# 查询最近登录的用户（30天内）
dsquery user -stalepwd 30

# 查询非活动账户
dsquery user -inactive 4

# 查询具有特定属性的计算机
dsquery computer -name "WEB*"

# 限制返回结果数量
dsquery user -limit 100
```

**结合dsget获取详细信息：**
```cmd
# 获取用户详细信息
dsquery user -name "admin" | dsget user -dn -samid -email -desc -disabled

# 获取组成员
dsquery group -name "Domain Admins" | dsget group -members

# 获取计算机信息
dsquery computer | dsget computer -dn -samid -desc -loc
```

### 2.3 wmic命令

wmic（WMI Command-line）提供丰富的系统管理信息。

**域信息查询：**
```cmd
# 查看域信息
wmic ntdomain get DomainControllerAddress,DomainName

# 查看计算机系统信息
wmic computersystem get Name,Domain,Manufacturer,Model

# 查看操作系统信息
wmic os get Caption,Version,OSArchitecture

# 查看用户账户
wmic useraccount get Name,SID,Domain,Disabled

# 查看组信息
wmic group get Name,SID,Domain

# 查看本地管理员组成员
wmic path win32_groupuser where (groupcomponent="win32_group.name=\"administrators\",domain=\"%computername%\"") get partcomponent
```

**远程查询：**
```cmd
# 远程查询计算机信息
wmic /node:192.168.1.10 /user:corp\admin /password:Pass123 computersystem get Name,Domain

# 远程查询进程
wmic /node:192.168.1.10 process get Name,ProcessId,CommandLine

# 远程查询服务
wmic /node:192.168.1.10 service get Name,DisplayName,State,StartMode
```

## 3. PowerShell侦察

### 3.1 PowerShell AD模块

Active Directory PowerShell模块提供了丰富的域管理命令。

**模块加载：**
```powershell

# 查看可用命令
Get-Command -Module ActiveDirectory
```

**域基础信息：**
```powershell
# 获取当前域信息
Get-ADDomain

# 获取域森林信息
Get-ADForest

# 获取域控制器
Get-ADDomainController
Get-ADDomainController -Discover

# 获取域功能级别
Get-ADDomain | Select-Object DomainMode
Get-ADForest | Select-Object ForestMode
```

**用户信息收集：**
```powershell
# 获取所有域用户
Get-ADUser -Filter * -Properties *

# 获取特定用户
Get-ADUser -Identity administrator -Properties *

# 获取启用的用户
Get-ADUser -Filter {Enabled -eq $true}

# 获取禁用的用户
Get-ADUser -Filter {Enabled -eq $false}

# 获取具有特定属性的用户
Get-ADUser -Filter * -Properties PasswordLastSet,LastLogonDate | 
    Where-Object {$_.PasswordLastSet -lt (Get-Date).AddDays(-90)}

# 搜索高权限用户
Get-ADUser -Filter * -Properties adminCount | Where-Object {$_.adminCount -eq 1}

# 获取用户组成员身份
Get-ADUser -Identity user1 -Properties MemberOf | Select-Object -ExpandProperty MemberOf

# 获取用户的SID History
Get-ADUser -Identity user1 -Properties sIDHistory
```

**组信息收集：**
```powershell
# 获取所有组
Get-ADGroup -Filter *

# 获取特定组
Get-ADGroup -Identity "Domain Admins"

# 获取组成员
Get-ADGroupMember -Identity "Domain Admins"
Get-ADGroupMember -Identity "Enterprise Admins"
Get-ADGroupMember -Identity "Schema Admins"

# 递归获取组成员（包含嵌套组）
Get-ADGroupMember -Identity "Domain Admins" -Recursive

# 获取高权限组
Get-ADGroup -Filter * -Properties adminCount | Where-Object {$_.adminCount -eq 1}
```

**计算机信息收集：**
```powershell
# 获取所有计算机
Get-ADComputer -Filter * -Properties *

# 获取域控制器
Get-ADComputer -Filter {OperatingSystem -like "*Server*" -and PrimaryGroupID -eq 516}

# 获取特定操作系统
Get-ADComputer -Filter {OperatingSystem -like "*Windows 10*"}
Get-ADComputer -Filter {OperatingSystem -like "*Server 2019*"}

# 获取启用的计算机
Get-ADComputer -Filter {Enabled -eq $true}

# 获取计算机最后登录时间
Get-ADComputer -Filter * -Properties LastLogonDate | 
    Sort-Object LastLogonDate -Descending

# 查找运行特定服务的计算机（通过SPN）
Get-ADComputer -Filter * -Properties servicePrincipalName | 
    Where-Object {$_.servicePrincipalName -like "*MSSQLSvc*"}
```

**组织单位收集：**
```powershell
# 获取所有OU
Get-ADOrganizationalUnit -Filter *

# 获取特定OU下的对象
Get-ADObject -Filter * -SearchBase "OU=IT,DC=corp,DC=local"

# 获取GPO链接
Get-ADOrganizationalUnit -Filter * -Properties gPLink
```

### 3.2 ADSI查询

ADSI（Active Directory Service Interfaces）是访问AD的底层接口，无需额外模块。

**基本ADSI查询：**
```powershell
# 创建AD搜索器
$searcher = [System.DirectoryServices.DirectorySearcher]::new()

# 设置搜索根
$searcher.SearchRoot = [System.DirectoryServices.DirectoryEntry]::new("LDAP://DC=corp,DC=local")

# 设置过滤器
$searcher.Filter = "(objectClass=user)"

# 执行搜索
$results = $searcher.FindAll()

# 遍历结果
foreach ($result in $results) {
    $result.Properties["samaccountname"]
}
```

**高级ADSI查询：**
```powershell
# 查询所有用户
$searcher = [adsisearcher]"(objectClass=user)"
$searcher.FindAll() | ForEach-Object {$_.Properties["samaccountname"]}

# 查询域管理员
$searcher = [adsisearcher]"(&(objectClass=user)(memberOf=CN=Domain Admins,CN=Users,DC=corp,DC=local))"
$searcher.FindAll()

# 查询具有SPN的账户（服务账户）
$searcher = [adsisearcher]"(&(objectClass=user)(servicePrincipalName=*))"
$searcher.FindAll()

# 查询计算机
$searcher = [adsisearcher]"(objectClass=computer)"
$searcher.PageSize = 1000
$searcher.FindAll()

# 查询特定属性
$searcher = [adsisearcher]"(objectClass=user)"
$searcher.PropertiesToLoad.Add("samAccountName")
$searcher.PropertiesToLoad.Add("mail")
$searcher.PropertiesToLoad.Add("telephoneNumber")
$searcher.FindAll()
```

**LDAP查询语法：**
```powershell
# 等于
"(samAccountName=admin)"

# 模糊匹配
"(samAccountName=adm*)"

# 存在属性
"(servicePrincipalName=*)"

# AND运算
"(&(objectClass=user)(adminCount=1))"

# OR运算
"(|(memberOf=CN=Domain Admins,...)(memberOf=CN=Enterprise Admins,...))"

# NOT运算
"(!(userAccountControl:1.2.840.113556.1.4.803:=2))"

# 位运算（禁用账户）
"(userAccountControl:1.2.840.113556.1.4.803:=2)"
```

### 4.5 自定义Cypher查询

**常用自定义查询：**

```cypher
// 查询所有域管理员
MATCH (u:User)-[:MemberOf]->(g:Group) WHERE g.name CONTAINS "DOMAIN ADMINS" RETURN u.name, g.name

// 查询到达特定用户的所有路径
MATCH p = shortestPath((u:User)-[*]->(t:User)) WHERE u.name = "USER@CORP.LOCAL" AND t.name = "ADMIN@CORP.LOCAL" RETURN p

// 查询所有Kerberoastable用户
MATCH (u:User) WHERE u.hasspn = true RETURN u.name, u.serviceprincipalnames

// 查询具有DCSync权限的用户
MATCH (u:User)-[:DCSync]->(d:Domain) RETURN u.name, d.name

// 查询拥有本地管理员权限的用户
MATCH (u:User)-[:AdminTo]->(c:Computer) RETURN u.name, c.name

// 查询会话信息
MATCH (u:User)-[:HasSession]->(c:Computer) RETURN u.name, c.name

// 查询ACL攻击路径
MATCH p = (u:User)-[:WriteDacl|WriteOwner|ForceChangePassword|GenericAll|GenericWrite|Owns]->(n) RETURN p

// 查询委派关系
MATCH (u:User)-[:AllowedToDelegate]->(c:Computer) RETURN u.name, c.name

// 查询SID History
MATCH (u:User) WHERE u.sidhistory IS NOT NULL RETURN u.name, u.sidhistory

// 查询密码不永不过期的用户
MATCH (u:User) WHERE u.pwdneverexpires = true RETURN u.name

// 查询GPO应用关系
MATCH (g:GPO)-[:GpLink]->(ou:OU) RETURN g.name, ou.name
```

### 4.6 攻击路径发现

**典型攻击路径分析：**

**场景1：从普通用户到域管理员**
```
User1 → AdminTo → Computer1
Computer1 → HasSession → DomainAdmin
```
发现：User1是Computer1的本地管理员，DomainAdmin在Computer1上有活跃会话，可以窃取凭证。

**场景2：ACL滥用路径**
```
User1 → WriteDacl → Group1
Group1 → MemberOf → Domain Admins
```
发现：User1对Group1有WriteDacl权限，Group1是Domain Admins成员，可以添加自己到Group1。

**场景3：委派攻击路径**
```
User1 → AllowedToDelegate → Computer1
Computer1 → TrustedToAuth → DomainController
```
发现：User1可以模拟任意用户访问Computer1，进一步攻击域控。

**BloodHound路径分析：**
1. 选择起始节点（当前用户）
2. 选择目标节点（域管理员）
3. 执行最短路径查询
4. 分析路径上的每个关系
5. 确定可利用的攻击技术

function Get-AllGroupMembership {
    param($UserName)
    $user = Get-ADUser -Identity $UserName -Properties MemberOf
    $groups = @()
    foreach ($groupDN in $user.MemberOf) {
        $group = Get-ADGroup -Identity $groupDN
        $groups += $group.Name
        # 递归获取父组
        $parentGroups = Get-ADGroup -Identity $groupDN -Properties MemberOf | Select-Object -ExpandProperty MemberOf
        foreach ($parent in $parentGroups) {
            $groups += (Get-ADGroup -Identity $parent).Name
        }
    }
    $groups | Sort-Object -Unique
}

function Invoke-DomainRecon {
    param(
        [string]$OutputPath = "C:\temp\recon"
    )
    
    # 创建输出目录
    New-Item -ItemType Directory -Path $OutputPath -Force
    
    Write-Host "[*] Starting Domain Reconnaissance..."
    
    # 域基础信息
    Write-Host "[*] Collecting Domain Information..."
    Get-ADDomain | Out-File "$OutputPath\domain.txt"
    Get-ADForest | Out-File "$OutputPath\forest.txt"
    
    # 域控制器
    Write-Host "[*] Collecting Domain Controllers..."
    Get-ADDomainController | Export-Csv "$OutputPath\dc.csv" -NoTypeInformation
    
    # 用户信息
    Write-Host "[*] Collecting Users..."
    Get-ADUser -Filter * -Properties * | Export-Csv "$OutputPath\users.csv" -NoTypeInformation
    
    # 组信息
    Write-Host "[*] Collecting Groups..."
    Get-ADGroup -Filter * | Export-Csv "$OutputPath\groups.csv" -NoTypeInformation
    
    # 域管理员
    Write-Host "[*] Collecting Domain Admins..."
    Get-ADGroupMember -Identity "Domain Admins" -Recursive | Export-Csv "$OutputPath\domain_admins.csv" -NoTypeInformation
    
    # 计算机信息
    Write-Host "[*] Collecting Computers..."
    Get-ADComputer -Filter * -Properties * | Export-Csv "$OutputPath\computers.csv" -NoTypeInformation
    
    # 服务账户
    Write-Host "[*] Collecting Service Accounts..."
    Get-ADUser -Filter {ServicePrincipalName -like "*"} -Properties ServicePrincipalName | Export-Csv "$OutputPath\service_accounts.csv" -NoTypeInformation
    
    # 信任关系
    Write-Host "[*] Collecting Trust Relationships..."
    Get-ADTrust -Filter * | Export-Csv "$OutputPath\trusts.csv" -NoTypeInformation
    
    # GPO信息
    Write-Host "[*] Collecting GPOs..."
    Get-GPO -All | Export-Csv "$OutputPath\gpos.csv" -NoTypeInformation
    
    Write-Host "[*] Reconnaissance Complete. Results saved to $OutputPath"
}

# 执行收集

function Invoke-AnalyzeRecon {
    param(
        [string]$DataPath = "C:\temp\recon"
    )
    
    Write-Host "=== Domain Reconnaissance Analysis ===" -ForegroundColor Green
    
    # 分析高价值目标
    Write-Host "`n[*] High Value Targets:" -ForegroundColor Yellow
    $admins = Import-Csv "$DataPath\domain_admins.csv"
    $admins | ForEach-Object { Write-Host "    - $($_.Name)" }
    
    # 分析服务账户
    Write-Host "`n[*] Service Accounts (Kerberoastable):" -ForegroundColor Yellow
    $svcAccounts = Import-Csv "$DataPath\service_accounts.csv"
    $svcAccounts | ForEach-Object { Write-Host "    - $($_.SamAccountName)" }
    
    # 分析计算机分布
    Write-Host "`n[*] Computer Distribution:" -ForegroundColor Yellow
    $computers = Import-Csv "$DataPath\computers.csv"
    $computers | Group-Object OperatingSystem | Sort-Object Count -Descending | 
        ForEach-Object { Write-Host "    - $($_.Name): $($_.Count)" }
    
    # 分析禁用账户
    Write-Host "`n[*] Disabled Accounts:" -ForegroundColor Yellow
    $users = Import-Csv "$DataPath\users.csv"
    $disabled = $users | Where-Object {$_.Enabled -eq "False"}
    Write-Host "    Total Disabled: $($disabled.Count)"
    
    # 分析长期未登录账户
    Write-Host "`n[*] Inactive Accounts (>90 days):" -ForegroundColor Yellow
    $inactive = $users | Where-Object {
        $_.LastLogonDate -and [datetime]$_.LastLogonDate -lt (Get-Date).AddDays(-90)
    }
    Write-Host "    Total Inactive: $($inactive.Count)"
}
