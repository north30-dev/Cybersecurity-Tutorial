# 靶场环境搭建

## 一、靶场概述与选择

### 1.1 什么是靶场

靶场（Cyber Range）是专门用于网络安全学习和实践的训练环境。它模拟了真实的网络环境、应用程序和系统，让学习者可以在合法、安全的环境中练习渗透测试、漏洞挖掘和防御技术。

靶场的核心价值：
- **合法性**：所有攻击行为都是授权的，不存在法律风险
- **安全性**：隔离环境不会影响真实生产系统
- **系统性**：从基础到高级，循序渐进的学习路径
- **实战性**：模拟真实场景，培养实际操作能力

### 1.2 靶场分类

#### 按部署方式分类

| 类型 | 特点 | 适用场景 | 代表平台 |
|------|------|----------|----------|
| 本地靶场 | 完全离线，自主可控 | 深度学习、自定义环境 | DVWA、Pikachu、vulhub |
| 云端靶场 | 开箱即用，环境统一 | 快速入门、CTF训练 | PortSwigger、HTB |
| 虚拟机靶场 | 真实系统模拟 | 综合渗透、红队演练 | VulnHub、OWASP BWA |

#### 按技能层次分类

| 层次 | 难度 | 适合人群 | 学习目标 |
|------|------|----------|----------|
| 入门级 | ⭐ | 零基础新手 | 了解漏洞原理，掌握基本工具 |
| 进阶级 | ⭐⭐ | 有基础的学习者 | 综合利用，绕过技巧 |
| 高级 | ⭐⭐⭐ | 安全从业者 | 复杂场景，组合攻击链 |
| 专家级 | ⭐⭐⭐⭐ | 安全专家 | APT模拟，红队对抗 |

### 1.3 靶场选择建议

#### 初学者路径（大一上学期）

```
第一阶段（1-4周）：DVWA → Pikachu → sqli-labs
第二阶段（5-8周）：PortSwigger Academy → upload-labs
第三阶段（9-12周）：HTB Starting Point → VulnHub Easy
```

#### 选择标准

1. **学习目标明确**：根据想学的漏洞类型选择对应靶场
2. **难度匹配**：不要一上来就挑战高难度，容易打击信心
3. **环境稳定**：选择维护活跃、文档完善的靶场
4. **社区支持**：有活跃社区可以交流讨论

## 二、DVWA环境搭建

DVWA（Damn Vulnerable Web Application）是最经典的Web安全学习靶场，涵盖了OWASP Top 10中的主要漏洞。

### 2.1 Docker部署（推荐）

#### 安装Docker环境

```bash
# Ubuntu/Debian系统
sudo apt update
sudo apt install -y docker.io docker-compose

# 启动Docker服务
sudo systemctl start docker
sudo systemctl enable docker

# 将当前用户加入docker组（免sudo）
sudo usermod -aG docker $USER
newgrp docker

# 验证安装
docker --version
docker-compose --version
```

#### 拉取并运行DVWA

```bash
# 拉取DVWA镜像
docker pull vulnerables/web-dvwa

# 运行DVWA容器
docker run -d -p 8080:80 --name dvwa vulnerables/web-dvwa

# 查看容器状态
docker ps | grep dvwa

# 访问测试
curl -I http://localhost:8080
```

#### 使用Docker Compose管理

创建 `docker-compose.yml` 文件：

```yaml
version: '3'
services:
  dvwa:
    image: vulnerables/web-dvwa
    container_name: dvwa
    ports:
      - "8080:80"
    environment:
      - DB_SERVER=db
    depends_on:
      - db
    restart: unless-stopped
  
  db:
    image: mysql:5.7
    container_name: dvwa-db
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: dvwa
    restart: unless-stopped
```

```bash
# 启动服务
docker-compose up -d

# 停止服务
docker-compose down

# 重启服务
docker-compose restart

# 查看日志
docker-compose logs -f dvwa
```

#### 环境准备

### 2.3 安全级别配置

DVWA提供四个安全级别，对应不同的防护强度：

| 级别 | 说明 | 防护措施 | 学习目标 |
|------|------|----------|----------|
| Low | 无防护 | 无任何过滤 | 了解漏洞原理 |
| Medium | 中等防护 | 基本过滤 | 学习绕过技巧 |
| High | 高级防护 | 强过滤 | 高级绕过技术 |
| Impossible | 不可能级别 | 完全防护 | 学习安全编码 |

#### 切换安全级别

**方式一：Web界面**
1. 登录DVWA（默认账号：admin/password）
2. 点击左侧 "DVWA Security"
3. 选择安全级别并提交

**方式二：修改配置文件**

```php
// config/config.inc.php
$_DVWA[ 'default_security_level' ] = 'low';  // 改为需要的级别
```

**方式三：Cookie修改**

```bash
# 使用curl设置安全级别
curl -b "security=low; PHPSESSID=xxx" http://localhost:8080/DVWA/

# 使用Burp Suite修改Cookie
# Cookie: security=low; PHPSESSID=xxx
```

### 2.4 漏洞模块介绍

DVWA包含以下漏洞模块：

#### 命令注入（Command Injection）

```php
// Low级别源码
<?php
if( isset( $_REQUEST[ 'Submit' ] ) ) {
    $target = $_REQUEST[ 'ip' ];
    $cmd = shell_exec( 'ping  -c 4 ' . $target );
    echo "<pre>{$cmd}</pre>";
}
?>
```

**漏洞原理**：直接拼接用户输入到系统命令
**利用方式**：`127.0.0.1; ls -la` 或 `127.0.0.1 && cat /etc/passwd`

#### SQL注入（SQL Injection）

```php
// Low级别源码
<?php
$id = $_REQUEST[ 'id' ];
$query  = "SELECT first_name, last_name FROM users WHERE user_id = '$id';";
$result = mysqli_query($GLOBALS["___mysqli_ston"], $query);
?>
```

**漏洞原理**：未过滤的用户输入直接拼接SQL语句
**利用方式**：`' OR '1'='1` 或 `' UNION SELECT user,password FROM users--`

#### XSS（Cross Site Scripting）

```php
// Reflected XSS - Low级别
<?php
if( array_key_exists( "name", $_GET ) && $_GET[ 'name' ] != NULL ) {
    echo '<pre>Hello ' . $_GET[ 'name' ] . '</pre>';
}
?>
```

**漏洞原理**：直接输出用户输入到HTML页面
**利用方式**：`<script>alert('XSS')</script>`

#### 文件上传（File Upload）

```php
// Low级别源码
<?php
if( isset( $_POST[ 'Upload' ] ) ) {
    $target_path = DVWA_WEB_PAGE_TO_ROOT . "hackable/uploads/";
    $target_path .= basename( $_FILES[ 'uploaded' ][ 'name' ] );
    if( move_uploaded_file( $_FILES[ 'uploaded' ][ 'tmp_name' ], $target_path ) ) {
        echo "<pre>{$target_path} succesfully uploaded!</pre>";
    }
}
?>
```

**漏洞原理**：无任何文件类型验证
**利用方式**：直接上传PHP WebShell

#### 暴力破解（Brute Force）

```php
// Low级别源码
<?php
if( isset( $_GET[ 'Login' ] ) ) {
    $user = $_GET[ 'username' ];
    $pass = $_GET[ 'password' ];
    $query = "SELECT * FROM `users` WHERE user = '$user' AND password = '$pass';";
    // 无登录失败次数限制，无验证码
}
?>
```

**漏洞原理**：无防护措施，可无限尝试
**利用方式**：使用Burp Suite Intruder进行爆破

## 三、Pikachu靶场

Pikachu是一个国产的Web安全靶场，漏洞类型丰富，中文界面友好。

### 3.1 环境搭建

#### Docker部署

```bash
# 拉取镜像
docker pull area39/pikachu

# 运行容器
docker run -d -p 8081:80 --name pikachu area39/pikachu

# 初始化数据库
docker exec -it pikachu /bin/bash
cd /var/www/html/pikachu
php install.php
exit
```

#### 源码安装

```bash
# 克隆仓库
cd /var/www/html
git clone https://github.com/zhuifengshaonianhanlu/pikachu.git

# 设置权限
chown -R www-data:www-data pikachu/
chmod -R 755 pikachu/

# 访问 http://localhost/pikachu/install.php 完成初始化
```

### 3.2 漏洞模块一览

| 模块 | 漏洞类型 | 难度 |
|------|----------|------|
| Bruteforce | 暴力破解 | ⭐ |
| XSS | 反射型/存储型/DOM型XSS | ⭐⭐ |
| CSRF | 跨站请求伪造 | ⭐⭐ |
| SQL-Inject | SQL注入（数字型/字符型） | ⭐⭐ |
| RCE | 命令执行/代码执行 | ⭐⭐ |
| File Inclusion | 文件包含（本地/远程） | ⭐⭐ |
| File Upload | 文件上传 | ⭐⭐ |
| SSRF | 服务端请求伪造 | ⭐⭐⭐ |
| XXE | XML外部实体注入 | ⭐⭐⭐ |
| Unsafe File Download | 任意文件下载 | ⭐⭐ |
| Over Permission | 越权访问 | ⭐⭐ |
| JWT | JWT安全问题 | ⭐⭐⭐ |

### 3.3 学习路径建议

```
Week 1-2: Bruteforce → XSS → CSRF
Week 3-4: SQL-Inject → RCE
Week 5-6: File Inclusion → File Upload
Week 7-8: SSRF → XXE → JWT
```

```
SQL injection → SQL injection UNION attacks → SQL injection exam
├── SQL injection vulnerability in WHERE clause

├── SQL injection UNION attack, finding column with text type
├── SQL injection UNION attack, retrieving data from other tables
├── SQL injection UNION attack, retrieving multiple columns
└── SQL injection exam (认证考试)
```

#### XSS路径

```
Cross-site scripting → XSS contexts → XSS exam
├── Reflected XSS into HTML context
├── Reflected XSS into HTML tag attribute
├── Reflected XSS into JavaScript string
├── Reflected XSS with angle brackets and double quotes HTML-encoded
├── Reflected XSS into a JavaScript string with angle brackets HTML encoded
└── Stored XSS into HTML context
```

Advanced topics:
├── HTTP request smuggling

## 八、总结

靶场是网络安全学习的最佳实践环境。通过系统化的靶场训练，可以：

1. **建立完整知识体系**：从原理到实践，全面理解漏洞
2. **培养实战能力**：在真实环境中锻炼渗透测试技能
3. **积累经验**：大量练习形成肌肉记忆和直觉
4. **准备认证**：为OSCP等认证考试打基础

**学习建议**：
- 不要贪多，每个漏洞都要深入理解
- 多动手实践，光看不练假把式
- 养成做笔记的习惯，方便复习总结
- 加入社区交流，向他人学习

**下一步**：
完成靶场环境搭建后，可以开始学习信息收集技术，这是渗透测试的第一步，也是最重要的一步。
