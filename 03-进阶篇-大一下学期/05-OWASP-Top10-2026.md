# OWASP Top 10 2026 详解

## 一、OWASP Top 10 2026概述

### 1.1 什么是OWASP Top 10

OWASP Top 10是开放Web应用安全项目（Open Web Application Security Project）发布的Web应用安全风险排行榜，每3-4年更新一次，代表了当前最关键的Web应用安全风险。OWASP Top 10被全球安全从业者、开发人员和企业广泛认可和应用。

### 1.2 OWASP Top 10 2026与2021对比

| 排名 | 2026 | 2021 | 变化说明 |
|------|------|------|----------|
| A01 | 访问控制失效 | 访问控制失效 | 保持不变 |
| A02 | 加密机制失效 | 加密机制失效 | 保持不变 |
| A03 | 注入攻击 | 注入攻击 | 扩展：新增软件供应链安全 |
| A04 | 不安全设计 | 不安全设计 | 保持不变 |
| A05 | 安全配置错误 | 安全配置错误 | 保持不变 |
| A06 | 易受攻击组件 | 易受攻击和过时的组件 | 简化名称 |
| A07 | 身份验证失败 | 身份识别和认证失败 | 简化名称 |
| A08 | 软件和数据完整性故障 | 软件和数据完整性故障 | 保持不变 |
| A09 | 日志监控失效 | 安全日志和监控失效 | 简化名称 |
| A10 | **异常处理不当（新增）** | 服务端请求伪造(SSRF) | 新增项 |

### 1.3 2026版本主要变化

```markdown
OWASP Top 10 2026 主要变化：

1. A03注入攻击扩展
   - 新增软件供应链安全内容
   - 强调CI/CD管道安全
   - 包含依赖包投毒攻击

2. A10新增：异常处理不当
   - 取代2021年的SSRF
   - 关注错误信息泄露
   - 关注异常处理导致的安全绕过

3. SSRR降级原因
   - SSRF攻击模式相对固定
   - 防护措施相对成熟
   - 异常处理问题更为普遍

4. 命名优化
   - 多个类别名称简化
   - 更直观表达风险本质
```

### 1.5 风险评估方法论

```
OWASP风险评分公式：

Risk = Likelihood × Impact

其中：
- Likelihood（可能性）：威胁因素 + 漏洞普遍性 + 漏洞可检测性
- Impact（影响）：技术影响 + 业务影响

评分维度：
1. 发生率（Incidence Rate）
2. 可利用性（Exploitability）
3. 普遍性（Prevalence）
4. 可检测性（Detectability）
5. 技术影响（Technical Impact）
```

## 二、A01:2026 - 访问控制失效（Broken Access Control）

### 2.1 漏洞概述

访问控制失效是指应用程序未能正确实施访问限制，导致用户可以访问未经授权的资源或执行未经授权的操作。这是连续多年排名第一的安全风险。

### 2.2 常见攻击类型

#### 2.2.1 水平越权

```markdown
水平越权：同级别用户之间的越权访问

场景：用户A可以访问用户B的私密数据

示例：
GET /api/user/1001/profile  → 用户A的信息
GET /api/user/1002/profile  → 用户B的信息（越权访问）

原因：
- 仅检查用户是否登录，未检查资源归属
- 使用可预测的资源ID
- 缺少资源级别的权限检查
```

#### 2.2.2 垂直越权

```markdown
垂直越权：低权限用户访问高权限功能

场景：普通用户执行管理员操作

示例：
POST /api/admin/deleteUser  → 普通用户执行删除操作
GET /admin/dashboard        → 普通用户访问管理后台

原因：
- 仅前端隐藏功能，后端未验证权限
- URL路径未做权限控制
- API接口缺少角色验证
```

#### 2.2.3 IDOR（不安全的直接对象引用）

```bash
# IDOR示例

# 正常请求
GET /api/orders/12345

# 修改ID访问他人订单
GET /api/orders/12346
GET /api/orders/12347

# 批量遍历
for i in range(10000, 20000):
    requests.get(f"/api/orders/{i}")
```

### 2.3 漏洞示例

**不安全的代码示例：**

```php
// 不安全：仅检查登录状态，未检查资源归属
function getUserProfile($userId) {
    if (!isAuthenticated()) {
        return "未登录";
    }
    // 直接根据ID查询，未验证权限
    $user = db_query("SELECT * FROM users WHERE id = ?", $userId);
    return $user;
}
```

```java
// 不安全：缺少权限验证
@GetMapping("/api/documents/{id}")
public Document getDocument(@PathVariable Long id) {
    return documentRepository.findById(id);
}
```

**安全的代码示例：**

```php
// 安全：验证资源归属
function getUserProfile($userId) {
    $currentUser = getCurrentUser();
    if (!$currentUser) {
        return "未登录";
    }
    // 验证资源归属
    if ($currentUser->id != $userId && !$currentUser->isAdmin()) {
        return "无权访问";
    }
    $user = db_query("SELECT * FROM users WHERE id = ?", $userId);
    return $user;
}
```

```java
// 安全：使用Spring Security注解
@PreAuthorize("hasRole('ADMIN') or #id == authentication.principal.id")
@GetMapping("/api/documents/{id}")
public Document getDocument(@PathVariable Long id) {
    return documentRepository.findById(id);
}

// 或在代码中验证
@GetMapping("/api/documents/{id}")
public Document getDocument(@PathVariable Long id, Principal principal) {
    Document doc = documentRepository.findById(id);
    if (!doc.getOwner().equals(principal.getName()) && 
        !hasRole("ADMIN")) {
        throw new AccessDeniedException("无权访问");
    }
    return doc;
}
```

### 2.4 防御措施

```markdown
访问控制防御措施：

1. 实施最小权限原则
   - 默认拒绝所有访问
   - 仅授予必要的最小权限

2. 服务端权限验证
   - 所有权限检查在服务端执行
   - 不依赖前端隐藏功能

3. 资源级别权限控制
   - 验证用户对特定资源的访问权限
   - 使用不可预测的资源标识符

4. 统一权限框架
   - 使用成熟的权限框架
   - 集中管理权限规则

5. 日志记录
   - 记录所有访问尝试
   - 监控异常访问模式
```

**权限验证中间件示例：**

```javascript
// Node.js权限验证中间件
function checkPermission(resource, action) {
    return async (req, res, next) => {
        const user = req.user;
        const resourceId = req.params.id;
        
        // 检查用户是否有权限
        const hasPermission = await permissionService.check({
            userId: user.id,
            resource: resource,
            resourceId: resourceId,
            action: action
        });
        
        if (!hasPermission) {
            return res.status(403).json({ error: '无权访问' });
        }
        
        next();
    };
}

// 使用示例
app.get('/api/documents/:id', 
    authenticate, 
    checkPermission('document', 'read'), 
    getDocument
);
```

## 三、A02:2026 - 加密机制失效（Cryptographic Failures）

### 3.1 漏洞概述

加密机制失效（原"敏感数据泄露"）主要涉及敏感数据的保护不足，包括传输中的数据、存储中的数据和处理中的数据。

### 3.2 常见问题

```markdown
加密机制常见问题：

1. 传输层问题
   - 使用HTTP而非HTTPS
   - 使用弱TLS版本（TLS 1.0/1.1）
   - 使用弱加密套件
   - 证书验证不当

2. 存储层问题
   - 明文存储密码
   - 使用弱哈希算法（MD5、SHA1）
   - 未使用盐值
   - 密钥管理不当

3. 应用层问题
   - 敏感数据缓存不当
   - 日志记录敏感数据
   - 错误信息泄露敏感数据
```

### 3.3 不安全的加密示例

```php
// 不安全：明文存储密码
function saveUser($username, $password) {
    db_query("INSERT INTO users VALUES (?, ?)", $username, $password);
}

// 不安全：使用MD5哈希
function hashPassword($password) {
    return md5($password);
}

// 不安全：使用无盐哈希
function hashPassword($password) {
    return sha256($password);
}
```

### 3.4 安全的加密示例

```php
// 安全：使用bcrypt哈希密码
function hashPassword($password) {
    // bcrypt自动处理盐值
    return password_hash($password, PASSWORD_BCRYPT, ['cost' => 12]);
}

function verifyPassword($password, $hash) {
    return password_verify($password, $hash);
}

// 安全：使用Argon2（更安全）
function hashPassword($password) {
    return password_hash($password, PASSWORD_ARGON2ID);
}
```

### 3.5 敏感数据加密

```java
// Java AES加密示例
import javax.crypto.Cipher;
import javax.crypto.spec.SecretKeySpec;
import javax.crypto.spec.GCMParameterSpec;
import java.util.Base64;

public class AESUtil {
    private static final String ALGORITHM = "AES/GCM/NoPadding";
    private static final int TAG_LENGTH = 128;
    
    public static String encrypt(String data, byte[] key, byte[] iv) throws Exception {
        SecretKeySpec secretKey = new SecretKeySpec(key, "AES");
        Cipher cipher = Cipher.getInstance(ALGORITHM);
        GCMParameterSpec spec = new GCMParameterSpec(TAG_LENGTH, iv);
        cipher.init(Cipher.ENCRYPT_MODE, secretKey, spec);
        byte[] encrypted = cipher.doFinal(data.getBytes());
        return Base64.getEncoder().encodeToString(encrypted);
    }
    
    public static String decrypt(String encryptedData, byte[] key, byte[] iv) throws Exception {
        SecretKeySpec secretKey = new SecretKeySpec(key, "AES");
        Cipher cipher = Cipher.getInstance(ALGORITHM);
        GCMParameterSpec spec = new GCMParameterSpec(TAG_LENGTH, iv);
        cipher.init(Cipher.DECRYPT_MODE, secretKey, spec);
        byte[] decrypted = cipher.doFinal(Base64.getDecoder().decode(encryptedData));
        return new String(decrypted);
    }
}
```

### 3.6 防御措施

```markdown
加密机制防御措施：

1. 传输安全
   - 强制使用HTTPS
   - 使用TLS 1.2或更高版本
   - 配置HSTS头部
   - 使用强加密套件

2. 密码存储
   - 使用bcrypt/Argon2/scrypt
   - 设置足够的工作因子
   - 永不存储明文密码

3. 敏感数据加密
   - 使用AES-256-GCM
   - 安全管理密钥
   - 使用硬件安全模块(HSM)

4. 密钥管理
   - 使用密钥管理服务
   - 定期轮换密钥
   - 分离开发和生产密钥
```

## 四、A03:2026 - 注入攻击（Injection）

### 4.1 漏洞概述

注入攻击是指不受信任的数据作为命令或查询的一部分发送给解释器，攻击者可以构造恶意数据欺骗解释器执行非预期的命令或访问非授权的数据。

### 4.2 注入类型

```markdown
常见注入类型：

1. SQL注入
   - 通过SQL语句获取或修改数据
   - 可能获取数据库权限

2. NoSQL注入
   - 针对MongoDB等NoSQL数据库
   - 使用JSON/BSON语法注入

3. 命令注入
   - 执行操作系统命令
   - 可能获取系统权限

4. LDAP注入
   - 修改LDAP查询语句
   - 绕过认证或获取数据

5. XPath注入
   - 修改XPath查询
   - 获取XML数据

6. 软件供应链注入（2026新增）
   - CI/CD管道注入
   - 依赖包投毒
```

### 4.3 SQL注入详解

#### 4.3.1 注入类型分类

```markdown
SQL注入类型：

1. 联合查询注入（Union-based）
   ' UNION SELECT username, password FROM users--

2. 布尔盲注（Boolean-based）
   ' AND 1=1--   (返回正常)
   ' AND 1=2--   (返回异常)

3. 时间盲注（Time-based）
   ' AND SLEEP(5)--
   ' AND BENCHMARK(10000000,SHA1('test'))--

4. 报错注入（Error-based）
   ' AND extractvalue(1,concat(0x7e,(SELECT password FROM users)))--

5. 堆叠查询（Stacked Queries）
   '; DROP TABLE users;--
```

#### 4.3.2 不安全的SQL代码

```php
// 不安全：直接拼接SQL
function getUser($id) {
    $sql = "SELECT * FROM users WHERE id = " . $id;
    return db_query($sql);
}

// 不安全：虽然使用引号，但未转义
function getUser($name) {
    $sql = "SELECT * FROM users WHERE name = '$name'";
    return db_query($sql);
}
```

#### 4.3.3 安全的SQL代码

```php
// 安全：使用PDO参数化查询
function getUser($id) {
    $stmt = $pdo->prepare("SELECT * FROM users WHERE id = :id");
    $stmt->bindParam(':id', $id, PDO::PARAM_INT);
    $stmt->execute();
    return $stmt->fetch();
}

// 安全：使用命名参数
function getUserByName($name) {
    $stmt = $pdo->prepare("SELECT * FROM users WHERE name = :name");
    $stmt->execute(['name' => $name]);
    return $stmt->fetch();
}
```


### 4.4 命令注入详解

#### 4.4.1 不安全的命令执行

```php
// 不安全：直接拼接命令

    system("ping -c 4 " . $host);
}

// 攻击payload

```

```python

import subprocess

def ping(host):
    os.system("ping -c 4 " + host)
    
def ping2(host):
    subprocess.call("ping -c 4 " + host, shell=True)
```

#### 4.4.2 安全的命令执行

```php
// 安全：使用escapeshellarg
function ping($host) {
    $host = escapeshellarg($host);
    system("ping -c 4 " . $host);
}

// 更安全：验证输入格式
function ping($host) {
    if (!filter_var($host, FILTER_VALIDATE_IP)) {
        throw new InvalidArgumentException("Invalid IP address");
    }
    system("ping -c 4 " . escapeshellarg($host));
}
```

```python
# 安全：不使用shell=True，使用列表传参
import subprocess

def ping(host):
    # 使用列表形式，避免shell解析
    result = subprocess.run(
        ["ping", "-c", "4", host],
        capture_output=True,
        text=True
    )
    return result.stdout

# 输入验证
import ipaddress

def ping_safe(host):
    try:
        ipaddress.ip_address(host)  # 验证IP格式
    except ValueError:
        raise ValueError("Invalid IP address")
    return subprocess.run(["ping", "-c", "4", host], capture_output=True)
```

## 五、A04:2026 - 不安全设计（Insecure Design）

### 5.1 漏洞概述

不安全设计是指系统架构和设计阶段引入的安全缺陷，这些缺陷无法通过完美的代码实现来修复。需要在设计阶段就考虑安全因素。

### 5.2 设计缺陷示例

```markdown
常见设计缺陷：

1. 缺乏安全建模
   - 未进行威胁建模
   - 未定义信任边界
   - 未考虑攻击场景

2. 业务逻辑缺陷
   - 优惠券可重复使用
   - 支付金额可篡改
   - 订单状态可伪造

3. 认证设计缺陷
   - 弱密码策略
   - 无账户锁定机制
   - 会话固定攻击

4. 授权设计缺陷
   - 缺乏资源级别授权
   - 角色权限设计不当
   - API权限控制缺失
```

### 5.3 安全设计原则

```markdown
安全设计原则：

1. 最小权限原则
   - 默认拒绝访问
   - 仅授予必要权限

2. 纵深防御
   - 多层安全控制
   - 不依赖单一防护

3. 安全默认
   - 默认配置安全
   - 安全开箱即用

4. 失败安全
   - 失败时保持安全状态
   - 不泄露敏感信息

5. 不信任原则
   - 验证所有输入
   - 不信任用户数据
```

### 5.4 威胁建模


```java
// 安全登录实现示例

@Service
public class AuthService {
    private static final int MAX_ATTEMPTS = 5;
    private static final long LOCK_TIME = 30 * 60 * 1000; // 30分钟
    
    @Autowired
    private UserRepository userRepository;
    
    @Autowired
    private LoginAttemptService attemptService;
    
    public AuthResult login(String username, String password, String ip) {
        // 检查IP是否被锁定
        if (attemptService.isBlocked(ip)) {
            return AuthResult.blocked("IP已被锁定");
        }
        
        // 检查用户是否被锁定
        User user = userRepository.findByUsername(username);
        if (user != null && user.isLocked()) {
            if (user.getLockTime() + LOCK_TIME > System.currentTimeMillis()) {
                return AuthResult.blocked("账户已被锁定");
            } else {
                user.unlock();
                userRepository.save(user);
            }
        }
        
        // 验证凭据
        if (user == null || !passwordEncoder.matches(password, user.getPassword())) {
            attemptService.recordFailedAttempt(ip, username);
            
            if (attemptService.getAttempts(username) >= MAX_ATTEMPTS) {
                user.lock();
                userRepository.save(user);
            }
            
            return AuthResult.failed("用户名或密码错误");
        }
        
        // 检查MFA
        if (user.isMfaEnabled()) {
            String mfaToken = generateMfaToken(user);
            return AuthResult.mfaRequired(mfaToken);
        }
        
        // 登录成功
        attemptService.clearAttempts(ip, username);
        String sessionToken = createSession(user);
        
        return AuthResult.success(sessionToken);
    }
}
```

```python
    @wraps(f)
    def decorated_function(*args, **kwargs):
        if 'user_id' not in session:
            return {'error': '未登录'}, 401
        return f(*args, **kwargs)
```

```python
from pydantic import BaseModel, EmailStr, constr, validator

class UserRequest(BaseModel):
    username: constr(min_length=3, max_length=50, regex=r'^[a-zA-Z0-9_]+$')
    password: constr(min_length=8, max_length=100)
    email: EmailStr
    age: int
    
    @validator('password')
    def validate_password(cls, v):
        if not any(c.islower() for c in v):
            raise ValueError('密码必须包含小写字母')
        if not any(c.isupper() for c in v):
            raise ValueError('密码必须包含大写字母')
        if not any(c.isdigit() for c in v):
            raise ValueError('密码必须包含数字')
        return v
```
