# 安全合规基础

## 概述

安全合规是网络安全的重要组成部分，涉及法律法规、行业标准、认证体系等多个方面。随着网络安全形势日益严峻，各国纷纷出台相关法律法规，企业也越来越重视安全合规工作。本章将系统介绍等保2.0、ISO 27001、GDPR以及国内主要网络安全法规，帮助读者建立安全合规的知识体系。

---

## 一、等保2.0基础

### 1.1 等级划分

网络安全等级保护（简称"等保"）是我国网络安全的基本制度，等保2.0于2019年12月1日正式实施，扩展了保护对象，强化了保护要求。

#### 等保等级划分原则

等级保护根据信息系统在国家安全、经济建设、社会生活中的重要程度，以及系统遭到破坏后对国家安全、社会秩序、公共利益以及公民、法人和其他组织的合法权益的危害程度，分为五个等级。

| 等级 | 对象重要性 | 受侵害后的危害程度 | 典型系统 |
|------|-----------|-------------------|----------|
| 第一级 | 一般系统 | 对公民、法人和其他组织的合法权益造成损害 | 小型企业网站、内部办公系统 |
| 第二级 | 重要系统 | 对公民、法人和其他组织的合法权益产生严重损害，或对社会秩序和公共利益造成损害 | 政府门户网站、一般企业业务系统 |
| 第三级 | 非常重要系统 | 对社会秩序和公共利益造成严重损害，或对国家安全造成损害 | 金融系统、电力系统、医疗系统 |
| 第四级 | 特别重要系统 | 对社会秩序和公共利益造成特别严重损害，或对国家安全造成严重损害 | 国家级金融系统、能源调度系统 |
| 第五级 | 极端重要系统 | 对国家安全造成特别严重损害 | 国家核心基础设施 |

#### 等保2.0保护对象扩展

等保2.0将保护对象从传统的信息系统扩展到：

1. **基础信息网络**：广域网、城域网、局域网等
2. **云计算平台**：公有云、私有云、混合云
3. **移动互联**：移动应用、移动终端
4. **物联网**：感知层、网络层、应用层
5. **工业控制系统**：SCADA、DCS、PLC等
6. **大数据平台**：数据采集、存储、处理、分析

### 1.2 测评流程

等保测评遵循"规定动作"流程，主要包括五个规定动作。

#### 等保工作流程

```
定级 → 备案 → 建设整改 → 等级测评 → 监督检查
```

**流程详解：**

| 阶段 | 主要工作 | 责任主体 | 输出成果 |
|------|----------|----------|----------|
| 定级 | 确定系统等级、编写定级报告 | 运营使用单位 | 定级报告 |
| 备案 | 向公安机关提交备案材料 | 运营使用单位 | 备案证明 |
| 建设整改 | 按等级要求进行安全建设 | 运营使用单位 | 安全建设方案 |
| 等级测评 | 委托测评机构进行测评 | 测评机构 | 测评报告 |
| 监督检查 | 公安机关定期监督检查 | 公安机关 | 检查记录 |

#### 定级备案流程

**步骤一：确定定级对象**
- 识别信息系统边界
- 确定系统业务功能
- 分析系统重要性

**步骤二：确定系统等级**
- 分析系统遭到破坏后的危害程度
- 依据定级指南确定等级
- 编写定级报告

**步骤三：专家评审**
- 组织专家对定级结果进行评审
- 第三级以上系统需专家评审
- 形成专家评审意见

**步骤四：公安机关备案**
- 准备备案材料
- 提交至属地公安机关网安部门
- 获取备案证明

#### 测评周期要求

| 等级 | 测评周期 | 测评要求 |
|------|----------|----------|
| 第一级 | 自主保护 | 无强制测评要求 |
| 第二级 | 建议两年一次 | 可自主检查或委托测评 |
| 第三级 | 每年一次 | 必须委托有资质的测评机构 |
| 第四级 | 每半年一次 | 必须委托有资质的测评机构 |
| 第五级 | 特殊要求 | 依照国家专门规定执行 |

### 1.3 安全要求

等保2.0的安全要求分为技术要求和管理要求两大类。

#### 技术要求

**安全物理环境**
- 物理位置选择：机房选址安全
- 物理访问控制：门禁、监控
- 防盗窃和防破坏：设备固定、标识
- 防火、防水、防潮、防静电
- 电力供应：UPS、备用电源
- 电磁防护：屏蔽、接地

**安全通信网络**
- 网络架构：分区分域、冗余设计
- 通信传输：加密传输、完整性保护
- 可信验证：网络设备可信验证

**安全区域边界**
- 边界防护：防火墙、入侵检测
- 访问控制：基于策略的访问控制
- 入侵防范：IDS/IPS
- 恶意代码防范：网关防病毒
- 安全审计：日志记录与分析

**安全计算环境**
- 身份鉴别：强认证、多因素认证
- 访问控制：最小权限、权限分离
- 安全审计：操作审计、日志保护
- 入侵防范：主机入侵检测
- 恶意代码防范：终端防护
- 数据完整性：数据校验
- 数据保密性：数据加密
- 数据备份恢复：备份策略

**安全管理中心**
- 系统管理：集中管理
- 审计管理：集中审计
- 安全管理：集中管控

#### 管理要求

**安全管理制度**
- 制定安全管理制度
- 发布和评审制度
- 执行和监督制度

**安全管理机构**
- 建立安全管理机构
- 明确岗位职责
- 配备安全人员

**安全管理人员**
- 人员录用：背景审查
- 人员离岗：权限回收
- 安全意识教育和培训
- 外部人员访问管理

**安全建设管理**
- 安全方案设计
- 安全产品采购
- 安全工程实施
- 安全测试验收

**安全运维管理**
- 环境维护
- 资产管理
- 介质管理
- 设备维护
- 漏洞和风险管理
- 网络和系统安全管理
- 恶意代码防范管理
- 配置变更管理
- 密码管理
- 备份与恢复管理

## 二、ISO 27001

### 2.1 标准概述

ISO/IEC 27001是国际标准化组织发布的信息安全管理体系标准，是目前国际上最权威的信息安全管理标准。

#### 标准体系结构

ISO 27000系列标准构成完整的信息安全管理体系：

| 标准编号 | 名称 | 内容 |
|----------|------|------|
| ISO 27000 | 概述与词汇 | 基本概念和术语定义 |
| ISO 27001 | 要求 | ISMS要求和认证依据 |
| ISO 27002 | 实践准则 | 安全控制措施指南 |
| ISO 27003 | 实施指南 | ISMS实施指导 |
| ISO 27004 | 度量 | 安全度量指标 |
| ISO 27005 | 风险管理 | 信息安全风险管理 |
| ISO 27006 | 认证机构要求 | 对认证机构的要求 |

#### ISO 27001核心要素

**PDCA循环模型：**
```
Plan（策划）→ Do（实施）→ Check（检查）→ Act（改进）
```

- **Plan**：建立信息安全管理体系，识别风险，制定风险处理计划
- **Do**：实施风险处理计划，执行控制措施
- **Check**：监控和评审ISMS的有效性
- **Act**：持续改进ISMS

### 2.2 认证流程

ISO 27001认证是企业建立和实施信息安全管理体系的重要证明。

#### 认证流程步骤

**阶段一：准备阶段**
1. 获取管理层支持
2. 确定认证范围
3. 组建项目团队
4. 制定工作计划

**阶段二：体系建设**
1. 现状调研与差距分析
2. 制定信息安全方针
3. 风险评估与风险处理
4. 编写体系文件
5. 体系试运行

**阶段三：内部审核**
1. 内审员培训
2. 内部审核实施
3. 不符合项整改
4. 管理评审

**阶段四：认证审核**
1. 选择认证机构
2. 提交认证申请
3. 第一阶段审核（文档审核）
4. 第二阶段审核（现场审核）
5. 不符合项整改
6. 获取认证证书

**阶段五：监督审核**
- 每年进行监督审核
- 三年进行再认证

#### 认证周期与费用

| 项目 | 说明 |
|------|------|
| 准备周期 | 6-12个月（视组织规模和基础） |
| 认证周期 | 2-3个月 |
| 证书有效期 | 3年 |
| 监督审核 | 每年一次 |
| 费用构成 | 咨询费 + 审核费 + 认证费 + 差旅费 |

### 2.3 关键控制点

ISO 27001附录A包含114个控制措施，分为14个控制域。

#### 核心控制域

| 控制域 | 控制数量 | 重点控制措施 |
|--------|----------|--------------|
| 信息安全方针 | 2 | 方针制定、评审 |
| 信息安全组织 | 7 | 组织架构、职责分配 |
| 人力资源安全 | 6 | 入职审查、培训、离职处理 |
| 资产管理 | 10 | 资产清单、分类标记 |
| 访问控制 | 14 | 访问策略、用户管理 |
| 密码学 | 2 | 加密策略、密钥管理 |
| 物理和环境安全 | 15 | 物理访问、设备安全 |
| 操作安全 | 13 | 变更管理、备份 |
| 通信安全 | 7 | 网络安全、数据传输 |
| 系统获取和开发 | 12 | 安全需求、开发安全 |
| 供应商关系 | 4 | 供应商管理 |
| 信息安全事件管理 | 7 | 事件响应、报告 |
| 业务连续性管理 | 5 | 业务影响分析、连续性计划 |
| 合规 | 8 | 法律合规、审计 |

## 三、GDPR数据保护

### 3.1 核心原则

通用数据保护条例（General Data Protection Regulation，GDPR）是欧盟于2018年5月25日实施的数据保护法规，对全球企业的数据处理活动产生深远影响。

#### GDPR核心原则

| 原则 | 英文 | 描述 |
|------|------|------|
| 合法、公平、透明 | Lawfulness, Fairness, Transparency | 数据处理必须有合法依据，公平对待数据主体 |
| 目的限制 | Purpose Limitation | 数据收集目的明确，不得用于其他目的 |
| 数据最小化 | Data Minimization | 只收集必要的数据 |
| 准确性 | Accuracy | 数据应准确并及时更新 |
| 存储限制 | Storage Limitation | 数据存储时间不超过必要期限 |
| 完整性和保密性 | Integrity and Confidentiality | 确保数据安全 |
| 可问责性 | Accountability | 数据控制者应能证明合规 |

#### 数据主体权利

| 权利 | 英文 | 描述 |
|------|------|------|
| 知情权 | Right to be Informed | 有权了解数据如何被处理 |
| 访问权 | Right of Access | 有权获取个人数据副本 |
| 更正权 | Right to Rectification | 有权更正不准确的数据 |
| 删除权 | Right to Erasure | 特定情况下有权要求删除数据 |
| 限制处理权 | Right to Restrict Processing | 特定情况下有权限制数据处理 |
| 数据可携带权 | Right to Data Portability | 有权以结构化格式获取数据 |
| 反对权 | Right to Object | 有权反对某些类型的数据处理 |
| 自动决策相关权利 | Rights related to Automated Decision | 有权不受纯自动决策的影响 |

### 3.2 合规要点

#### 合法处理依据

数据处理必须至少满足以下一项合法依据：

1. **同意**：数据主体明确同意
2. **合同履行**：履行合同所必需
3. **法律义务**：遵守法律义务
4. **重大利益**：保护数据主体或他人重大利益
5. **公共利益**：执行公共利益任务
6. **合法利益**：控制者或第三方的合法利益

#### 关键角色与职责

| 角色 | 英文 | 职责 |
|------|------|------|
| 数据控制者 | Data Controller | 决定数据处理目的和方式 |
| 数据处理者 | Data Processor | 代表控制者处理数据 |
| 数据保护官 | Data Protection Officer | 监督合规、提供建议 |
| 数据主体 | Data Subject | 个人数据所属的个人 |

#### 合规实施要点

**数据保护影响评估（DPIA）**
- 高风险处理活动前必须进行
- 评估处理对数据主体权利的影响
- 制定风险缓解措施

**数据泄露通知**
- 72小时内通知监管机构
- 高风险情况需通知数据主体
- 记录所有数据泄露事件

**跨境数据传输**
- 充分性决定国家可直接传输
- 其他情况需保障措施
- 标准合同条款、绑定企业规则等

## 四、国内法规

### 4.1 网络安全法

《中华人民共和国网络安全法》于2017年6月1日正式实施，是我国网络安全领域的基础性法律。

#### 主要内容框架

| 章节 | 内容要点 |
|------|----------|
| 网络安全支持与促进 | 标准制定、人才培养、产业发展 |
| 网络运行安全 | 等级保护、安全监测、应急预案 |
| 网络信息安全 | 个人信息保护、违法信息处置 |
| 监测预警与应急处置 | 安全监测、预警发布、应急处置 |
| 法律责任 | 违法行为的法律责任 |

#### 关键义务

**网络运营者义务：**
- 实施网络安全等级保护制度
- 采取技术措施防范网络攻击
- 制定网络安全应急预案
- 收集使用个人信息遵循合法正当必要原则
- 发生安全事件时及时报告

**关键信息基础设施运营者额外义务：**
- 设置专门安全管理机构和负责人
- 对重要系统和数据库进行容灾备份
- 每年至少进行一次安全检测评估
- 采购网络产品和服务需安全审查

### 4.2 数据安全法

《中华人民共和国数据安全法》于2021年9月1日正式实施，建立了数据分类分级保护制度。

#### 核心制度

**数据分类分级保护：**
- 根据数据在经济社会发展中的重要程度
- 根据数据遭到篡改、破坏、泄露或者非法获取后对国家安全、公共利益或者公民、组织合法权益的危害程度
- 对数据实行分类分级保护

**重要数据保护：**
- 识别重要数据目录
- 加强重要数据保护
- 数据安全风险评估
- 数据出境安全评估

**数据安全管理制度：**
- 建立全流程数据安全管理制度
- 组织开展数据安全教育培训
- 采取相应技术措施保障数据安全

### 4.3 个人信息保护法

《中华人民共和国个人信息保护法》于2021年11月1日正式实施，是我国首部个人信息保护专门法律。

#### 核心规定

**个人信息处理规则：**
- 处理个人信息应当遵循合法、正当、必要和诚信原则
- 应当具有明确、合理的目的
- 应当与处理目的直接相关
- 采取对个人权益影响最小的方式

**敏感个人信息：**
- 生物识别、宗教信仰、特定身份、医疗健康、金融账户、行踪轨迹等
- 处理敏感个人信息需取得单独同意
- 需告知处理的必要性和对个人权益的影响

**个人信息跨境提供：**
- 需具备法定条件
- 通过安全评估或认证
- 订立标准合同

## 五、合规实践建议

### 5.1 企业合规建设路径

**第一阶段：合规现状评估**
- 识别适用的法律法规和标准
- 进行合规差距分析
- 确定合规优先级

**第二阶段：合规体系建设**
- 建立合规管理组织架构
- 制定合规管理制度
- 实施技术控制措施

**第三阶段：合规运行维护**
- 持续监控合规状态
- 定期进行合规审计
- 及时整改合规问题

**第四阶段：合规持续改进**
- 跟踪法规变化
- 优化合规措施
- 提升合规水平

### 5.2 合规工作要点

| 工作领域 | 关键任务 | 输出成果 |
|----------|----------|----------|
| 制度建设 | 制定安全管理制度、流程 | 制度文件体系 |
| 组织建设 | 建立安全管理组织、配备人员 | 组织架构、岗位职责 |
| 技术建设 | 实施安全技术措施 | 技术方案、配置文档 |
| 运营管理 | 日常安全运营、事件响应 | 运营记录、事件报告 |
| 审计评估 | 定期安全审计、合规评估 | 审计报告、评估报告 |

### 5.3 常见合规问题与解决方案

| 问题 | 解决方案 |
|------|----------|
| 合规意识不足 | 开展合规培训，建立合规文化 |
| 制度执行不到位 | 加强监督检查，建立考核机制 |
| 技术能力不足 | 引入专业服务，提升技术能力 |
| 资源投入不足 | 争取管理层支持，合理规划预算 |
| 法规理解偏差 | 跟踪法规解读，咨询专业机构 |

## 六、总结

安全合规是网络安全工作的重要保障，涉及多个法律法规和标准体系。通过本章学习，读者应当掌握：

1. **等保2.0基础**：理解等级划分、测评流程和安全要求
2. **ISO 27001**：掌握标准体系、认证流程和关键控制点
3. **GDPR数据保护**：了解核心原则、数据主体权利和合规要点
4. **国内法规**：熟悉网络安全法、数据安全法、个人信息保护法的主要内容
5. **合规实践**：掌握企业合规建设路径和工作要点

在实际工作中，应当根据组织实际情况，识别适用的法规标准，建立有效的合规管理体系，确保持续满足合规要求。
