## 三、技术面试准备

### 3.1 Web安全高频题

#### 基础概念题

**Q1：请介绍OWASP Top 10**

参考答案：
OWASP Top 10是OWASP组织发布的Web应用十大安全风险，每3-4年更新一次。2021版本包括：

1. **A01:2021 - Broken Access Control（访问控制失效）**
   - 越权访问、垂直/水平权限绕过
   
2. **A02:2021 - Cryptographic Failures（加密失败）**
   - 敏感数据明文存储、弱加密算法
   
3. **A03:2021 - Injection（注入）**
   - SQL注入、命令注入、LDAP注入
   
4. **A04:2021 - Insecure Design（不安全设计）**
   - 缺乏安全设计、威胁建模不足
   
5. **A05:2021 - Security Misconfiguration（安全配置错误）**
   - 默认配置、错误信息泄露
   
6. **A06:2021 - Vulnerable and Outdated Components（易受攻击和过时的组件）**
   - 使用有漏洞的第三方组件
   
7. **A07:2021 - Identification and Authentication Failures（身份识别和认证失败）**
   - 弱密码、会话管理不当
   
8. **A08:2021 - Software and Data Integrity Failures（软件和数据完整性失败）**
   - 不安全的反序列化、供应链攻击
   
9. **A09:2021 - Security Logging and Monitoring Failures（安全日志和监控失败）**
   - 日志不足、监控缺失
   
10. **A10:2021 - Server-Side Request Forgery（SSRF）**
    - 服务端请求伪造

**Q2：SQL注入的原理、类型和防御方法**

参考答案：

**原理：**
SQL注入是由于应用程序对用户输入的数据没有进行充分的过滤和验证，直接将用户输入拼接到SQL语句中执行，导致攻击者可以构造恶意输入改变原有SQL语句的逻辑，从而获取、修改或删除数据库中的数据。

**类型：**
1. **联合查询注入**：通过UNION合并多个SELECT语句的结果
2. **报错注入**：利用数据库报错信息提取数据
3. **布尔盲注**：通过页面响应差异判断条件真假
4. **时间盲注**：通过响应时间差异判断条件真假
5. **堆叠查询注入**：执行多条SQL语句
6. **二次注入**：恶意数据存储后被其他功能调用触发

**防御方法：**
1. **使用参数化查询/预编译语句**（最有效）
2. **输入验证**：白名单验证，过滤特殊字符
3. **最小权限**：数据库账户使用最小必要权限
4. **输出编码**：对输出数据进行编码
5. **WAF防护**：部署Web应用防火墙
6. **安全编码规范**：遵循安全编码最佳实践

**Q3：XSS的原理、类型和防御方法**

参考答案：

**原理：**
XSS（跨站脚本攻击）是由于Web应用程序对用户输入的数据没有进行充分的过滤和编码，直接输出到页面中，导致攻击者可以注入恶意脚本代码，在其他用户的浏览器中执行，从而窃取用户信息、执行恶意操作等。

**类型：**
1. **反射型XSS**：恶意脚本通过URL参数传递，服务器处理后返回
2. **存储型XSS**：恶意脚本存储在服务器，每次访问都会执行
3. **DOM型XSS**：前端JavaScript直接操作DOM导致

**防御方法：**
1. **输出编码**：根据输出上下文进行HTML/JavaScript/URL编码
2. **输入验证**：对用户输入进行白名单验证
3. **CSP（内容安全策略）**：限制脚本来源
4. **HttpOnly Cookie**：防止脚本访问敏感Cookie
5. **使用安全框架**：现代框架通常有内置的XSS防护

**Q4：CSRF的原理和防御方法**

参考答案：

**原理：**
CSRF（跨站请求伪造）是攻击者诱导用户在已登录的目标网站执行非预期操作。攻击者构造恶意请求，利用用户已认证的身份，在用户不知情的情况下发送请求，完成攻击。

**防御方法：**
1. **CSRF Token**：为每个请求生成随机Token，验证请求中的Token
2. **SameSite Cookie**：设置Cookie的SameSite属性
3. **验证Referer/Origin头**：检查请求来源
4. **二次验证**：关键操作要求二次确认（密码、验证码）
5. **自定义请求头**：使用X-Requested-With等自定义头

**Q5：SSRF的原理和防御方法**

参考答案：

**原理：**
SSRF（服务端请求伪造）是攻击者利用服务器发起请求的能力，让服务器访问攻击者指定的内网资源或外部资源，从而探测内网、访问内部服务、读取敏感文件等。

**利用场景：**
- 探测内网IP和端口
- 访问内网服务（Redis、MySQL等）
- 读取本地文件（file://协议）
- 绕过IP限制访问外部资源

**防御方法：**
1. **URL白名单**：只允许访问预定义的合法URL
2. **协议限制**：只允许HTTP/HTTPS协议
3. **IP验证**：解析域名后验证IP是否为内网地址
4. **禁用重定向**：或对重定向进行验证
5. **网络隔离**：限制服务器访问内网的权限

#### 深度技术题

**Q6：如何绕过WAF进行SQL注入**

参考答案：

常见WAF绕过技术：

1. **编码绕过**
   - URL编码：`%27`、`%2527`
   - Unicode编码：`%u0027`
   - HTML实体编码

2. **大小写混合**
   - `SeLeCt`、`UnIoN`

3. **注释符绕过**
   - 内联注释：`/*!50000select*/`
   - 普通注释：`sel/**/ect`

4. **等价函数替换**
   - `concat()` → `concat_ws()`
   - `group_concat()` → `group_concat(distinct ...)`
   - `sleep()` → `benchmark()`

5. **特殊字符替代**
   - 空格替代：`%20`、`%09`、`%0a`、`/**/`
   - 引号替代：使用`char()`函数

6. **分块传输编码**
   - 使用Transfer-Encoding: chunked

7. **参数污染**
   - 重复参数名：`id=1&id=2' union select`

**Q7：如何进行SQL注入的盲注**

参考答案：

盲注是在页面没有直接显示数据或错误信息的情况下，通过页面响应差异或响应时间来推断数据。

**布尔盲注流程：**

```sql
-- 1. 判断注入点
?id=1' and 1=1 --+  (正常)
?id=1' and 1=2 --+  (异常)

-- 2. 判断数据库长度
?id=1' and length(database())>5 --+

-- 3. 逐字符猜解数据库名
?id=1' and ascii(substr(database(),1,1))>100 --+

-- 4. 猜解表名数量
?id=1' and (select count(*) from information_schema.tables where table_schema=database())>5 --+

-- 5. 逐字符猜解表名
?id=1' and ascii(substr((select table_name from information_schema.tables where table_schema=database() limit 0,1),1,1))>100 --+
```

**时间盲注流程：**

```sql
-- 1. 判断注入点
?id=1' and sleep(5) --+

-- 2. 条件延时
?id=1' and if(length(database())>5,sleep(5),0) --+

-- 3. 逐字符猜解
?id=1' and if(ascii(substr(database(),1,1))>100,sleep(5),0) --+
```

**自动化工具：**
- SQLMap：`sqlmap -u "url" --technique=B`（布尔盲注）
- SQLMap：`sqlmap -u "url" --technique=T`（时间盲注）

### 3.2 内网安全高频题

**Q8：内网渗透的基本流程**

参考答案：

内网渗透通常遵循以下流程：

```
外网突破 → 信息收集 → 权限提升 → 横向移动 → 权限维持 → 痕迹清理
```

**详细步骤：**

1. **外网突破**
   - Web漏洞利用
   - 钓鱼攻击
   - VPN/边界设备漏洞

2. **信息收集**
   - 系统信息：`systeminfo`、`whoami`
   - 网络信息：`ipconfig`、`netstat`
   - 域信息：`net user`、`net group`
   - 敏感文件搜索

3. **权限提升**
   - 提权漏洞利用
   - 配置错误利用
   - 服务漏洞利用

4. **横向移动**
   - 凭证窃取（Mimikatz）
   - 哈希传递（PTH）
   - 票据传递（PTT）
   - 远程执行（PsExec、WMI）

5. **权限维持**
   - 创建隐藏账户
   - 计划任务
   - 注册表后门

**Q9：Windows常用的提权方法**

参考答案：

1. **内核漏洞提权**
   - 检测：`systeminfo`对比漏洞库
   - 利用：PrintNightmare、CVE-2021-36934等

2. **服务配置错误**
   - 不安全的服务权限
   - 不安全的服务路径
   - 服务DLL劫持

3. **计划任务**
   - 检查可写的计划任务
   - 替换执行文件

4. **AlwaysInstallElevated**
   - MSI安装提权

5. **Token窃取**
   - Token impersonation
   - Potato系列攻击

6. **UAC绕过**
   - 各种UAC绕过技术

**Q10：域环境中常见的攻击手法**

参考答案：

1. **信息收集**
   - 域用户枚举
   - 域控制器定位
   - 组策略收集

2. **凭证窃取**
   - LSASS内存转储
   - SAM数据库转储
   - NTDS.dit提取

3. **哈希传递（PTH）**
   - 利用NTLM哈希直接认证
   - `mimikatz "sekurlsa::pth /user:admin /domain:contoso /ntlm:hash"`

4. **票据传递（PTT）**
   - 黄金票据：伪造TGT
   - 白银票据：伪造服务票据

5. **Kerberoasting**
   - 请求服务票据
   - 离线破解票据获取密码

6. **DCSync**
   - 模拟域控制器复制
   - 导出所有域用户哈希

### 3.3 安全开发高频题

**Q11：什么是安全开发生命周期（SDL）**

参考答案：

安全开发生命周期（SDL）是将安全实践融入软件开发全过程的方法论，核心理念是"安全左移"。

**微软SDL七个阶段：**
1. 培训：安全意识培训
2. 需求：确定安全需求
3. 设计：威胁建模
4. 实现：安全编码
5. 验证：安全测试
6. 发布：安全审查
7. 响应：安全响应

**关键活动：**
- 威胁建模（STRIDE）
- 安全代码审查
- 静态/动态分析
- 渗透测试

**Q12：如何防止敏感信息泄露**

参考答案：

**存储安全：**
- 敏感数据加密存储
- 使用安全的哈希算法存储密码
- 密钥安全存储和管理

**传输安全：**
- 使用HTTPS加密传输
- 证书正确配置
- 敏感参数不在URL中传递

**日志安全：**
- 日志脱敏处理
- 不记录敏感信息
- 日志访问控制

**错误处理：**
- 生产环境不显示详细错误
- 自定义错误页面
- 错误信息不包含敏感数据

**配置安全：**
- 敏感配置加密
- 环境变量管理
- 配置文件不提交版本控制

**Q13：JWT安全注意事项**

参考答案：

**安全问题：**
1. **算法篡改**：将算法改为"none"绕过验证
2. **弱密钥**：使用弱密钥可被暴力破解
3. **信息泄露**：Payload未加密，敏感信息泄露
4. **重放攻击**：缺少过期时间或一次性使用机制

**安全实践：**
1. 使用强密钥（至少256位）
2. 使用安全的算法（RS256优于HS256）
3. 不在Payload中存储敏感信息
4. 设置合理的过期时间
5. 实现Token刷新机制
6. 验证Token的签名算法
7. 使用白名单验证算法

### 3.4 AI安全高频题

**Q14：AI模型面临哪些安全威胁**

参考答案：

1. **对抗样本攻击**
   - 通过精心设计的扰动使模型误分类
   - 影响图像识别、语音识别等

2. **数据投毒**
   - 在训练数据中注入恶意样本
   - 影响模型学习结果

3. **模型窃取**
   - 通过查询接口窃取模型
   - 获取模型参数或功能

4. **成员推断攻击**
   - 判断某样本是否在训练集中
   - 隐私泄露风险

5. **模型逆向攻击**
   - 从模型输出推断训练数据特征
   - 敏感信息泄露

**Q15：如何保护AI系统安全**

参考答案：

1. **对抗鲁棒性**
   - 对抗训练
   - 输入预处理
   - 检测对抗样本

2. **数据安全**
   - 数据清洗和验证
   - 差分隐私
   - 数据访问控制

3. **模型保护**
   - 模型加密
   - 限制查询频率
   - 输出扰动

4. **系统安全**
   - 安全的API设计
   - 访问控制
   - 审计日志
